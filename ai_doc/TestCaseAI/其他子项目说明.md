## 一、snow-testcase

### 1.1 项目定位

AI 测试用例生成实验项目，包含两类内容：
1. **ai-testcase/**：基于 LangChain 的 AI Agent 原型实验
2. **doc-based-test-generator/**：Claude Code 技能包（可直接调用的测试用例生成工具）

### 1.2 ai-testcase 模块

**技术栈：** Python + LangChain + OpenAI API

**文件结构：**

| 文件 | 说明 |
|------|------|
| `langchain-demo.py` | LangChain 框架集成 Demo |
| `langchain-rag-demo.py` | LangChain + RAG 检索增强 Demo |
| `robot1.py ~ robot5.py` | 多机器人协作实验（逐步演进版本） |
| `dataAnalysisAgent.py` | 数据分析 Agent |
| `documentGenerateAgent.py` | 文档生成 Agent |
| `multitranslateAgent.py` | 多语言翻译 Agent |
| `codeReviewSystem.py` | 代码审查系统 |
| `intelligentCustomer.py` | 智能客服 Demo |
| `demo1.py / demo.py` | 基础功能 Demo |
| `requirements.txt` | Python 依赖 |

**功能特点：**
- 基于 LangChain 的 Agent/Chain 模式
- 探索 AI 在测试用例生成领域的应用
- 多个 Agent 角色的协作实验

### 1.3 doc-based-test-generator（Claude Code 技能包）

**技术栈：** Claude Code Skill + Playwright MCP

**定位：** 一个结构化的 Claude Code 技能包，用于基于文档自动生成测试用例。

**文件结构：**
```
doc-based-test-generator/
├── SKILL.md           # 技能主文件 - 定义测试用例生成流程
├── skill.json         # OpenSkills 规范元数据
├── docs/
│   ├── quick-start.md       # 快速开始指南
│   ├── usage-guide.md       # 详细使用指南
│   └── save-mechanism.md    # 保存机制说明
├── references/
│   ├── functional-testcases-standard.md   # 功能测试用例标准
│   ├── api-testcases-standard.md          # API 测试用例标准
│   ├── performance-testcases-standard.md  # 性能测试用例标准
│   └── automation-testcases-standard.md   # 自动化测试用例标准
└── assets/
    └── README.md           # 资源说明
```

**业务流程：**
1. 用户输入：文档路径 + 测试需求
2. 技能自动分析文档结构
3. 参考标准模板生成测试用例
4. 支持 Playwright MCP 交互式操作验证

**特色：**
- 结构化的测试用例标准参考（功能/API/性能/自动化四种类型）
- 支持多种文档输入格式
- 输出结构化测试用例

### 1.4 基础设施

```
.claude/
├── settings.local.json       # Claude Code 本地设置
└── skills/                   # 本地技能包
    ├── doc-coauthoring/      # 文档协作技能
```

## 二、auto-test-framework

### 2.1 项目定位

企业级 Python 自动化测试框架，三种自动化测试模式统一管理。

**技术栈：** Python + Pytest + Selenium + Requests + Loguru

### 2.2 框架架构

```
auto-test-framework/
├── Base/               # 基础组件层（框架核心）
├── PageObject/         # 页面对象层（测试对象封装）
├── TestSuits/          # 测试用例层
├── Config/             # 配置文件
├── Data/               # 测试数据
├── Driver/             # 浏览器驱动
├── Reports/            # 测试报告
├── ExtTools/           # 扩展工具
├── RunMain/            # 运行入口
├── Log/                # 运行日志
└── conftest.py         # Pytest 全局配置
```

### 2.3 基础组件（Base/）

| 模块 | 说明 |
|------|------|
| `baseAutoWeb.py` | Web 自动化基类（Selenium WebDriver 封装） |
| `baseAutoHttp.py` | HTTP 接口自动化基类（requests 封装） |
| `baseAutoClient.py` | Client 自动化基类（TCP/UDP socket 封装） |
| `baseData.py` | 数据驱动（测试数据管理） |
| `baseExcel.py` | Excel 数据处理（openpyxl） |
| `baseYaml.py` | YAML 配置处理 |
| `baseLogger.py` | 日志封装（loguru） |
| `baseUtils.py` | 通用工具函数 |
| `basePath.py` | 路径管理 |
| `baseContainer.py` | 容器管理（Docker？） |
| `baseGuiRun.py` | GUI 运行器 |
| `baseSendEmail.py` | 邮件发送 |

### 2.4 项目对象（PageObject/）

| 模块 | 被测系统 | 测试类型 |
|------|---------|---------|
| `p01_client_xsglxt/` | 销售管理系统 | Client 自动化 |
| `p02_web_gjxt/` | 公文管理系统 | Web 自动化（登录/文章/文件） |
| `p03_http_gjxt/` | 公文管理系统 | HTTP 接口自动化（登录/文章/文件） |
| `project02_auto_test/` | 另一个被测系统 | 待完善 |

### 2.5 测试用例（TestSuits/）

| 套件 | 测试类型 | 状态 |
|------|---------|------|
| `p01_client_xsglxt/` | Client | 需要完善 |
| `p02_web_gjxt/test_web_gjxt.py` | Web | 有基础用例 |
| `p03_http_gjxt/test_http_gjxt.py` | HTTP | 有基础用例 |
| `project02_auto_test/test_case01.py` | 综合 | 单个用例 |

### 2.6 运行方式

```bash
# 命令行运行
python RunMain/run.py

# GUI 运行器
python RunMain/runClient.py

# 浏览器驱动初始化位置
Driver/  (Chrome/Firefox/Edge 驱动)
```

## 四、chat_ai_test

### 4.1 项目定位

多智能体（Multi-Agent）协作的测试用例生成助手，支持 SSE 流式输出。

**技术栈：** FastAPI + React + AutoGen 0.7.5

### 4.2 项目结构

```
chat_ai_test/
├── backend/
│   ├── main.py                    # FastAPI 入口文件
│   └── app/
│       ├── api/
│       │   ├── routes.py          # 路由注册
│       │   └── endpoints/
│       │       └── chat.py        # 聊天 API 端点
│       ├── core/
│       │   └── config.py          # 应用配置
│       ├── models/
│       │   └── __init__.py        # Pydantic 数据模型
│       └── services/
│           ├── autogen_service.py # AutoGen 多智能体服务
│           └── __init__.py
├── frontend/
│   ├── src/
│   │   └── services/
│   │       └── chatService.js     # 聊天服务（SSE 客户端）
│   ├── vite.config.js              # Vite 配置（代理）
│   └── package.json
├── example/
│   ├── multi_agent.py             # 多智能体示例
│   ├── openai_chat.py             # OpenAI 聊天示例
│   ├── openai_model_client.py     # OpenAI 模型客户端
│   ├── openai_singleton.py        # OpenAI 单例模式
│   └── autogen_chat.py            # AutoGen 聊天示例
├── start.sh / start-dev.sh        # 启动脚本
└── README.md
```

### 4.3 多智能体协作流程

```
用户输入测试需求
  ↓
测试用例设计师 (primary)
  ├── 根据需求生成详细的测试用例
  └── 输出到流
      ↓
测试用例评审师 (critic)
  ├── 系统性评审
  ├── 指出问题
  └── 提出改进建议
      ↓
用户评审 (user)
  ├── 评审结果
  ├── 决定接受或重新设计
  └── 输出最终结果
```

### 4.4 API 端点

```
GET  /health               # 健康检查
POST /api/chat/stream      # SSE 流式聊天（多智能体协作）
POST /api/chat/message     # 非流式聊天
```

### 4.5 核心实现

- **AutoGen 集成**：使用 RoundRobinGroupChat 实现智能体轮流输出
- **流式输出**：使用 SourceMatchTermination 控制结束条件，run_stream() 实现
- **前端**：React + Vite，参考 Gemini/智谱风格 UI，Markdown 渲染

## 五、lingma_ai_project

### 5.1 项目定位

基于 AutoGen + DeepSeek 的多代理聊天应用，与 chat_ai_test 功能类似但使用 Vue 3 前端。

**技术栈：** FastAPI + Vue 3 + ant-design-vue + AutoGen 0.7.5 + DeepSeek

### 5.2 项目结构

```
lingma_ai_project/
├── backend/
│   ├── main.py                    # FastAPI 入口
│   └── app/
│       ├── config.py              # DeepSeek 配置
│       ├── file_handler.py        # 文件处理器
│       ├── sse_handler.py         # SSE 事件处理器
│       └── team_manager.py        # AutoGen 团队管理
├── frontend/                      # Vue 3 + ant-design-vue
├── example/                       # 示例代码（同 chat_ai_test）
├── README.md / QUICKSTART.md / TESTING_GUIDE.md
└── 启动脚本（start.sh / start-dev.sh）
```

### 5.3 API 端点

```
GET  /health              # 健康检查
POST /api/session         # 创建会话
POST /api/chat            # 发送消息
POST /api/upload          # 文件上传
GET  /api/stream/{id}     # SSE 流式响应
```

### 5.4 核心实现

- **DeepSeek 集成**：通过 OpenAI-compatible API 接入
- **SSE 实时流式输出**：服务端 EventSource 推送
- **会话管理**：session_id 关联请求和响应
- **消息格式**：事件类型包括 connected / processing / AgentResponse / done

### 5.5 与 chat_ai_test 对比

| 维度 | chat_ai_test | lingma_ai_project |
|------|-------------|-------------------|
| 前端框架 | React | Vue 3 (ant-design-vue) |
| LLM 模型 | 通用（SiliconFlow） | DeepSeek |
| 最新状态 | 迭代较新（2025 最近修改） | 较旧版本 |
| 功能完整度 | 较完善 | 基础功能 |

## 六、interface-automation

### 6.1 项目定位

接口自动化测试学习项目，从基础到进阶的 8 天练习。

**技术栈：** Python + Pytest + Allure

### 6.2 学习路径

```
day1 → 作业：接口测试基础概念
day2 → 作业：接口测试方法
day3 → （内容待完善）
day4 → 接口封装实践（Event 活动管理 API）
day5 → （作业）
day6 → Pytest 基础（setup/teardown/fixture/skip/parametrize）
day7 → 登录接口测试 + JSON 数据驱动
day8 → 完整业务流程（登录 → 购物车 → 下单）
```

### 6.3 Day8 完整用例结构

```
day8/
├── case/
│   ├── conftest.py          # Pytest fixture（session/token 管理）
│   ├── test_login.py        # 登录测试
│   ├── test_shopping.py     # 购物流程测试
│   └── test_process.py      # 完整业务流程测试
├── data/
│   └── login_json.json      # 登录测试数据
├── db/
│   ├── __init__.py
│   └── db.py                # MySQL 数据库操作
├── interface/
│   ├── memberInterface.py   # 会员接口封装
│   ├── cartInterface.py     # 购物车接口封装
│   └── orderInterface.py    # 订单接口封装
├── utils/
│   ├── SendMethod.py        # HTTP 请求封装
│   ├── OperationJson.py     # JSON 文件操作
│   └── Jsonpath.py          # JSON 路径提取
└── case/Report/             # Allure 报告
```

### 6.4 技术要点

- Pytest fixture 管理登录 token/session
- 数据驱动：JSON 测试数据文件
- Allure 报告生成
- 接口分层封装：interface → case → data
- 数据库操作支持
