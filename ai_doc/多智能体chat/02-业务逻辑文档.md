# 业务逻辑文档 - AI 测试用例协作助手

## 1. 核心业务流程

### 1.1 多智能体协作流程

```
┌──────────────────────────────────────────────────────────────────┐
│                        用户输入需求                                │
│  (例如："设计登录功能的测试用例，字段：手机号、密码...")            │
└──────────────────────────┬───────────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────────────┐
│                      AutoGenService.get_response()               │
│                                                                   │
│  ┌─────────────────────────────────────────────────────────────┐  │
│  │              RoundRobinGroupChat 轮询调度                     │  │
│  │                                                              │  │
│  │  第1步: primary_agent (测试用例设计师) ◄──────────────┐      │  │
│  │    → 运用等价类、边界值、判定表生成测试用例集              │      │  │
│  │    → 输出表格格式的结构化用例                             │      │  │
│  │                                                          │      │  │
│  │  第2步: critic_agent (测试用例评审师) ◄──────────────── │      │  │
│  │    → 从6大维度系统性评审                                  │      │  │
│  │    → 输出评审报告（评分+缺陷清单+改进建议）                │      │  │
│  │                                                          │      │  │
│  │  第3步: user_proxy (用户评审) ◄─────────────────────── │      │  │
│  │    → 查看评审结果，决定接受/拒绝/修改                      │      │  │
│  │    → 接受则触发终止条件                                    │      │  │
│  │                                                          │      │  │
│  │  当 user_proxy 输出完毕 → SourceMatchTermination 触发     │      │  │
│  └─────────────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────────────┐
│                     SSE 流式输出到前端                             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐              │
│  │ 设计师输出    │→│ 评审师输出    │→│ 用户确认     │              │
│  │ (蓝色标识)    │  │ (橙色标识)    │  │ (绿色标识)   │              │
│  └─────────────┘  └─────────────┘  └─────────────┘              │
└──────────────────────────────────────────────────────────────────┘
```

### 1.2 智能体角色职责

#### 1.2.1 测试用例设计师（primary_agent）

**角色定位：** 拥有 10 年以上经验的高级测试架构师

**输入要求（用户需提供 A/B/C 至少一项）：**
- **A 类 - 需求文档/用户故事：** 功能描述、业务规则、界面原型、接口定义
- **B 类 - 技术规格：** 输入/输出字段、数据类型、取值范围、约束条件、状态转换图
- **C 类 - 上下文信息：** 目标用户群、使用场景、集成依赖、已知风险、历史缺陷

**测试设计方法论（必须使用的 6 种技术）：**

| 技术 | 适用场景 | 示例 |
|------|---------|------|
| 等价类划分 | 输入数据分类 | 有效等价类 + 无效等价类 |
| 边界值分析 | 边界附近取值 | min-1, min, min+1, max-1, max, max+1 |
| 判定表 | 多条件组合依赖 | 条件桩 + 动作桩，避免全组合爆炸 |
| 状态转换 | 有状态流转的系统 | 合法转换 + 非法转换全覆盖 |
| 场景法 | 端到端业务流程 | 基本流 + 备选流 + 异常流 |
| 错误猜测 | 经验驱动的风险测试 | 空值、超长、特殊字符、并发操作 |

**输出格式标准（表格形式）：**

```
| 编号 | 测试级别 | 前置条件 | 测试数据 | 操作步骤 | 预期结果 | 实际结果 | 优先级 |
| LOGIN-001 | 功能 | 账号未锁定 | 手机=138xx, 密码=abc123 | 1.输入 2.点击登录 | 跳转首页 | (留空) | P0 |
```

**覆盖准则（设计完成后自检）：**
1. **功能覆盖：** Happy Path ≥1 条，备选每分支≥1 条，异常每类≥1 条
2. **边界覆盖：** 数值型：min/min-1/min+1/max/max+1/max-1/0/负数/空值；字符型：min长度/max长度/空/超长/特殊字符/Unicode
3. **组合覆盖：** 多条件依赖用判定表或正交表
4. **状态覆盖：** 所有合法+非法状态转换
5. **非功能覆盖：** 并发/大数据量/SQL注入/XSS/越权/浏览器兼容/移动端

**优先级标准：**
- **P0（核心阻塞）：** 无替代路径、涉及资金/安全/核心数据、阻塞后续测试
- **P1（重要）：** 常用功能、重要业务场景
- **P2（一般）：** 边缘情况、体验优化
- **P3（建议）：** 极少场景、建议性

**迭代机制：** 设计完成后提供覆盖率自查清单。需求不明确时反问 2-3 个关键问题。

#### 1.2.2 测试用例评审师（critic_agent）

**角色定位：** 10 年以上经验的测试用例评审专家

**六大评审维度：**

| 维度 | 检查点 | 不通过后果 |
|------|--------|-----------|
| **1. 完整性** | 是否覆盖所有功能点/需求条目？是否缺少异常、边界、性能/安全场景？前置条件、测试数据、预期结果是否缺失或模糊？ | 核心场景遗漏 |
| **2. 准确性** | 预期结果是否与需求一致？操作步骤是否导致歧义或不可执行？测试数据是否合法？ | 误判/无效用例 |
| **3. 可执行性** | 步骤是否原子化、可重现？是否依赖未说明的环境配置？是否存在无法获取的数据或外部依赖？ | 无法执行 |
| **4. 覆盖率** | 等价类/边界值是否覆盖？状态转换路径？组合条件方法是否合理？非功能用例是否包含？ | 测试盲区 |
| **5. 优先级合理性** | P0 是否对应核心阻塞项？是否所有用例都是 P0 或 P3 的失衡情况？是否与业务风险匹配？ | 测试策略失效 |
| **6. 一致性** | 用例命名风格、步骤粒度是否统一？相同预期结果描述是否一致？优先级标准是否在全集中一致？ | 维护成本高 |

**严重程度分级：**
- **高：** 缺失核心场景、预期结果错误、导致测试无法进行
- **中：** 边界覆盖不足、步骤模糊、优先级不当
- **低：** 描述不一致、可读性优化、非关键场景缺失

**评审报告输出结构：**
1. **概览：** 总用例数、评审时间、整体评分（满分100）、评分等级
2. **缺陷清单：** 缺陷ID、用例编号、所属维度、问题描述、严重程度、改进建议
3. **覆盖率热力图（可选）：** 正常场景/异常场景/边界值/性能/安全的覆盖比例
4. **优秀实践亮点（可选）**
5. **风险提示：** 可能遗漏的线上风险
6. **改进建议汇总：** 必须修改项 + 建议优化项

#### 1.2.3 用户评审（user_proxy）

**角色定位：** 用户决策代理

**决策逻辑：**
1. 查看 critic 智能体的评审结果
2. 决定是否接受评审意见
3. 接受 → 回复"评审通过，采纳所有建议"
4. 不接受 → 说明具体拒绝理由或需要修改的地方
5. 需要重新设计 → 说明具体需求

### 1.3 终止条件逻辑

```
用户评审(user_proxy)输出完成
        │
        ▼
SourceMatchTermination(["user"]) 检测到 user 源的消息
        │
        ▼
RoundRobinGroupChat 停止协作
        │
        ▼
发送 data: [DONE] 信号
        │
        ▼
前端标记流式完成，将流式内容存入消息列表
```

## 2. 前端业务逻辑

### 2.1 状态管理

```
App.jsx 组件状态
├── messages: Array          ← 已完成的对话消息列表
│   ├── { role: 'user', content: "用户问题" }
│   └── { role: 'assistant', content: "AI完整回复" }
├── isLoading: Boolean       ← 是否正在加载
├── streamingContent: String ← 当前流式输出内容（实时更新）
└── currentAgent: String     ← 当前正在输出的智能体名称
```

### 2.2 流式数据处理逻辑

```
sendMessage(message, onChunk)  ← chatService.js
│
├── 1. fetch POST /api/chat/stream
├── 2. 获取 response.body reader
├── 3. 循环读取 ReadableStream chunks
│   ├── 解码 buffer
│   ├── 按 '\n\n' 分割 SSE 消息
│   ├── 解析 'data: ' 前缀的 JSON
│   │   ├── 检测 "【测试用例设计师】" → 切换智能体标记
│   │   ├── 检测 "【测试用例评审师】" → 切换智能体标记
│   │   ├── 检测 "【用户评审】" → 切换智能体标记
│   │   ├── 检测 "--- 用户评审完成 ---" → 结束标记
│   │   └── 普通内容 → 追加到当前智能体的 agentResponse
│   └── 调用 onChunk(chunk) 回调
│
└── 4. 流式完成后，将完整内容 push 到 messages
```

### 2.3 消息渲染逻辑

```
MessageList.jsx
│
├── 空状态（messages.length === 0 && !isLoading）
│   └── 显示引导页面 + 快捷建议
│
├── 已完成的每条消息（messages.map）
│   ├── role === 'user' → 用户消息样式（右对齐，蓝色气泡）
│   └── role === 'assistant' → AI消息样式（左对齐）
│       └── parseAgentContent(content) → 按角色分割渲染
│           ├── 【测试用例设计师】→ 蓝色徽章
│           ├── 【测试用例评审师】→ 橙色徽章
│           └── 【用户评审】→ 绿色徽章
│
└── 正在加载（isLoading）
    ├── streamingContent 非空 → 流式渲染
    │   └── parseAgentContent(streamingContent) → 实时更新
    └── streamingContent 为空 → 打字机动画
```

### 2.4 输入处理逻辑

```
InputArea.jsx
│
├── Enter（不按 Shift）→ 提交表单 → handleSubmit
├── Shift + Enter → 换行
├── 输入时自动调整 textarea 高度（max 200px）
├── 空内容或 isLoading → 禁用发送按钮
└── 提交后清空输入框
```

## 3. 后端业务逻辑

### 3.1 AutoGenService 单例模式

```python
class AutoGenService:
    _instance = None          # 类级别单例

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance._initialized = False
        return cls._instance

    def __init__(self):
        if self._initialized:
            return              # 阻止重复初始化
        # ... 初始化模型客户端、智能体、团队 ...
        self._initialized = True
```

**设计意图：** 确保整个应用生命周期内只创建一个 model_client 和 agent 团队实例。模型客户端初始化较慢（建立 HTTP 连接池），且 AI 调用成本高，单例模式避免重复创建。

### 3.2 流式响应生成逻辑

```python
async def get_response(self, message: str):
    stream = self.team.run_stream(task=message)
    current_agent = None

    async for item in stream:
        if 智能体切换（item.source ≠ current_agent）:
            yield SSE格式的角色标记 【测试用例设计师】

        if isinstance(item, ModelClientStreamingChunkEvent):
            for char in item.content:
                yield SSE格式的逐字符内容    # 逐字符实现打字机效果

        elif isinstance(item, TextMessage):
            if item.source == "user":
                yield "用户评审完成"标记
                break                       # 终止协作

        elif isinstance(item, TaskResult):
            yield data: [DONE]              # 任务完成
```

**关键实现细节：**
- **逐字符输出：** 拆解 `ModelClientStreamingChunkEvent` 的内容为字符级粒度，使前端呈现真正的打字机效果
- **角色标记注入：** 在智能体切换时向前端发送 `【角色名】` 标记行，前端据此解析和着色
- **SSE 转义：** 字符内容中 `\`、`"`、`\n` 需要转义以符合 SSE 的 JSON 格式

### 3.3 非流式响应生成逻辑

```python
async def get_full_response(self, message: str) -> str:
    result = await self.team.run(task=message)
    # 合并所有 TextMessage，加上角色标记
    # 格式：【测试用例设计师】\n内容\n\n【测试用例评审师】\n内容
```

### 3.4 错误处理逻辑

```
├── 网络/API 错误 → 捕获 Exception
│   ├── 打印 traceback 到控制台
│   └── 向前端发送 SSE 格式的错误消息
│       yield SSE: data: {"error": "错误描述"}
│       yield SSE: data: [DONE]
│
└── 前端错误处理
    ├── HTTP 非 200 状态 → throw Error
    ├── SSE JSON 解析失败 → console.error()，跳过
    └── catch 块 → 显示友好错误消息
```

## 4. 辅助项目业务逻辑

### 4.1 ai_test_learn（测试用例智能生成系统）

**增强功能：**
- **多模态图片分析：** 支持上传思维导图/流程图/界面截图，通过 `Qwen/Qwen2-VL-72B-Instruct` 视觉模型提取测试需求

## 5. 数据流图

### 5.1 请求处理数据流

```
用户 → InputArea → App.handleSendMessage()
                         │
                         ▼
                   chatService.sendMessage(message, onChunk)
                         │
                         │ POST /api/chat/stream
                         ▼
                   FastAPI chat_stream()
                         │
                         ▼
                   AutoGenService.get_response(message)
                         │
                         │ team.run_stream(task=message)
                         ▼
                   RoundRobinGroupChat
                         │
                         ├── primary_agent (设计用例)
                         ├── critic_agent (评审用例)
                         └── user_proxy (确认结果)
                         │
                         ▼
                   AsyncGenerator[SSE chunks]
                         │
                         ▼
               StreamingResponse(generate(), media_type="text/event-stream")
                         │
                         │ 逐字符 SSE 推送
                         ▼
                   chatService.js ReadableStream
                         │
                         │ onChunk 回调
                         ▼
                    App.jsx streamingContent 状态更新
                         │
                         ▼
                   MessageList.jsx 实时渲染
```

### 5.2 流式数据转换管道

```
AutoGen ModelClientStreamingChunkEvent (token级别)
        │
        ▼
AutoGenService 逐字符拆解 + 角色标记注入
        │
        ▼
SSE 格式: data: {"content": "字"}  (字符级别)
        │
        ▼
chatService.js 解析 SSE → onChunk(字符) 回调
        │
        ▼
App.jsx 解析角色标记 → 分流到 fullResponse / agentResponse
        │
        ▼
MessageList.jsx parseAgentContent → 按角色渲染彩色徽章 + Markdown
```
