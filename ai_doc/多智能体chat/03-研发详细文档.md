## 2. 关键技术实现

### 2.1 AutoGen 流式输出实现

**核心 API：`team.run_stream(task=message)`**

`run_stream()` 返回一个异步生成器（`AsyncGenerator`），在 AI 模型生成过程中持续产出以下类型的事件：

| 事件类型 | 说明 | 处理方式 |
|---------|------|---------|
| `ModelClientStreamingChunkEvent` | 模型的逐 token 输出 | 提取 `item.content`，逐字符发送给前端 |
| `TextMessage` | 智能体的完整消息输出 | 检测 `item.source` 判断智能体身份 |
| `TaskResult` | 团队协作完成 | 发送 `[DONE]` 标记 |

**流式粒度控制：**

```python
# autogen_service.py: 逐字符输出实现
for char in content:
    escaped_char = char.replace('\\', '\\\\').replace('"', '\\"').replace('\n', '\\n')
    yield f"data: {{\"content\": \"{escaped_char}\"}}\n\n"
```

### 2.2 SSE 协议实现

**后端（StreamingResponse）：**

```python
return StreamingResponse(
    generate(),
    media_type="text/event-stream",
    headers={
        "Cache-Control": "no-cache",          # 禁用缓存
        "Connection": "keep-alive",           # 长连接
        "X-Accel-Buffering": "no",           # 禁用 Nginx 缓冲
    }
)
```

**前端（ReadableStream）：**

```javascript
// chatService.js
const response = await fetch('/api/chat/stream', { ... })
const reader = response.body.getReader()
const decoder = new TextDecoder()
let buffer = ''

while (true) {
    const { done, value } = await reader.read()
    if (done) break
    buffer += decoder.decode(value, { stream: true })
    const messages = buffer.split('\n\n')      // SSE 边界分割
    buffer = messages.pop() || ''
    for (const msg of messages) {
        const lines = msg.split('\n')
        if (line.startsWith('data: ')) {
            const parsed = JSON.parse(line.slice(6))
            onChunk(parsed.content)            // 回调通知
        }
    }
}
```

### 2.4 单例模式

```python
class AutoGenService:
    _instance = None
    _initialized = False

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

    def __init__(self):
        if self._initialized:
            return
        # ... 初始化（仅执行一次）
        self._initialized = True
```

**设计权衡：** 单例避免了多次初始化 AI 模型客户端（昂贵的 HTTP 连接池），但牺牲了并发隔离性。多个并发请求共享同一个团队实例，在高并发场景下可能出现状态冲突。

## 7. 错误处理

### 7.1 后端错误处理

| 场景 | 异常类型 | 处理方式 | 前端表现 |
|------|---------|---------|---------|
| AI API 调用失败 | Exception | 捕获后发送 SSE 错误消息 | 显示错误提示 |
| 模型超时 | TimeoutError | 发送错误消息 | 显示"抱歉，发生了错误" |
| 请求参数错误 | ValidationError | FastAPI 自动返回 422 | HTTP 错误状态 |
| 服务器内部错误 | Exception | traceback 打印 + SSE 错误 | 显示错误提示 |

### 7.2 前端错误处理

| 场景 | 处理方式 |
|------|---------|
| HTTP 非 200 响应 | catch Error → 显示"抱歉，发生了错误" |
| SSE JSON 解析失败 | console.error，跳过该消息 |
| 流式读取中断 | catch Error → 清理状态 |
| 加载中重复发送 | isLoading 判断阻止 |

## 8. 性能与安全

### 8.1 性能考虑

- **单例模式：** 避免重复创建 AI 模型客户端（代价：HTTP 连接池初始化）
- **逐字符 SSE：** 虽然保障了实时性，但增加了网络传输量（每个字符一个 SSE frame）
- **Vite 开发代理：** 减少跨域配置的复杂度
- **react-markdown：** 在客户端渲染 Markdown，节省后端计算资源

### 8.2 安全考虑

- **CORS 配置：** 仅允许 `localhost:3000` 和 `127.0.0.1:3000` 跨域
- **API 密钥：** 存储在后端 `.env` 文件中，不暴露给前端
- **Pydantic 验证：** FastAPI 自动验证请求数据格式

### 8.3 当前安全风险（需修复）

| 风险 | 文件位置 | 说明 | 修复建议 |
|------|---------|------|---------|
| API 密钥硬编码 | `config.py:7` | `AUTOGEN_API_KEY` 存在内嵌默认值 | 删除默认值，强制从 `.env` 读取 |
| API 密钥硬编码 | `example/openai_chat.py:10` | API 密钥直接写在源码中 | 移除或改用环境变量 |
| API 密钥硬编码 | `example/openai_model_client.py:16` | 同上 | 移除或改用环境变量 |

### 9.2 生产环境部署注意事项

- 前端构建：`npm run build` → 产出 dist/ 目录
- 后端启动：推荐使用 `gunicorn -k uvicorn.workers.UvicornWorker main:app`
- 反向代理：建议使用 Nginx 前置代理，配置 SSE 支持
- 环境变量：通过系统环境变量或 `.env` 文件配置
- 端口：默认后端 8000，前端 3000（开发）/ 80/443（生产）

### 9.3 Nginx 配置参考（生产环境）

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_buffering off;          # 关闭缓冲（SSE 必需）
        proxy_cache off;              # 关闭缓存（SSE 必需）

    location / {
        root /path/to/frontend/dist;

    }
```
