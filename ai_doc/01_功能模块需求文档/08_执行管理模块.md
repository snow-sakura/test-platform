# 08 - 执行管理模块

## 模块定位
测试计划与测试执行的完整管理流程，支持创建测试计划、自动生成 TestRun、逐用例执行并追踪状态，是测试活动的核心执行引擎。

---

## 后端需求

### 8.1 测试计划模型（TestPlan）

| 字段 | 类型 | 说明 |
|------|------|------|
| name | CharField(200) | 计划名称 |
| description | TextField | 计划描述 |
| projects | M2M(Project) | 关联项目 |
| version | FK(Version) | 关联版本 |
| creator | FK(User) | 创建者 |
| assignees | M2M(User) | 指派给 |
| is_active | BooleanField | 是否激活 |
| created_at | DateTimeField | 创建时间 |
| updated_at | DateTimeField | 更新时间 |

### 8.2 测试执行模型（TestRun）

| 字段 | 类型 | 说明 |
|------|------|------|
| name | CharField(200) | 执行名称 |
| description | TextField | 执行描述 |
| test_plan | FK(TestPlan) | 关联测试计划 |
| project | FK(Project) | 关联项目 |
| version | FK(Version) | 关联版本 |
| testcases | M2M(TestCase) | 测试用例（通过 TestRunCase） |
| assignee | FK(User) | 执行人 |
| creator | FK(User) | 创建者 |
| status | CharField(20) | untested/in_progress/completed/blocked |
| started_at | DateTimeField | 开始时间 |
| completed_at | DateTimeField | 完成时间 |
| due_date | DateTimeField | 截止日期 |

### 8.3 执行用例模型（TestRunCase）

| 字段 | 类型 | 说明 |
|------|------|------|
| test_run | FK(TestRun) | 关联执行 |
| testcase | FK(TestCase) | 关联用例 |
| status | CharField(20) | untested/passed/failed/blocked/retest |
| priority | CharField(20) | low/medium/high/critical |
| actual_result | TextField | 实际结果 |
| comments | TextField | 备注 |
| defects | JSONField | 关联缺陷 ID 列表 |
| elapsed_time | DurationField | 执行耗时 |
| executed_by | FK(User) | 执行者 |
| executed_at | DateTimeField | 执行时间 |
| **约束** | unique_together | test_run + testcase |

### 8.4 执行历史模型（TestRunCaseHistory）

| 字段 | 类型 | 说明 |
|------|------|------|
| run_case | FK(TestRunCase) | 关联执行用例 |
| status | CharField(20) | 执行状态 |
| actual_result | TextField | 实际结果 |
| comments | TextField | 备注 |
| executed_by | FK(User) | 执行者 |
| executed_at | DateTimeField | 执行时间 |

### 8.5 API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/runs/plans/` | GET/POST | 计划列表/创建（自动为每个项目创建 TestRun） |
| `/api/runs/plans/{id}/` | GET/PUT/DELETE | 详情（含 TestRuns 嵌套）/更新/删除 |
| `/api/runs/plans/testcases_by_projects/` | GET | 根据项目获取测试用例列表 |
| `/api/runs/runs/` | GET/POST | 执行列表/创建 |
| `/api/runs/runs/{id}/` | GET/PUT/DELETE | 详情/更新/删除 |
| `/api/runs/run_cases/` | GET/POST | 用例执行列表/创建 |
| `/api/runs/run_cases/{id}/` | GET/PUT/DELETE | 详情/更新/删除 |
| `/api/runs/run_cases/{id}/update_status/` | PATCH | 更新执行状态（自动记录历史） |
| `/api/runs/run_cases/{id}/history/` | GET | 获取执行历史 |
| `/api/runs/history/` | GET | 历史列表 |

### 8.6 核心业务逻辑
- **自动创建 TestRun**：创建 TestPlan 时，自动为每个关联项目生成一个 TestRun
- **进度统计**：TestRun.progress_stats 动态计算 total/passed/failed/blocked/untested 数量
- **执行历史**：每次更新状态自动写入 TestRunCaseHistory
- **执行者追踪**：记录每次执行的操作人和时间

## 前端需求

### 8.7 页面组件

| 页面 | 路由 | 功能 |
|------|------|------|
| ExecutionListView.vue | `/ai-generation/executions` | 测试计划列表，支持状态筛选、批量删除 |
| ExecutionDetailView.vue | `/ai-generation/executions/:id` | 执行详情，展示 TestRun 列表和统计 |
| ExecutionList.vue | `/ai-generation/executions/list` | 执行记录列表（占位符） |

### 8.8 列表页功能
- **筛选**：项目筛选、状态筛选
- **表格列**：计划名称、关联项目、版本、状态、执行进度条、创建者、操作
- **操作**：新建计划、编辑、删除、批量删除

### 8.9 详情页功能
- **TestRun 列表**：每个项目一个 TestRun，含统计卡片（总数/通过/失败/阻塞/未测）
- **用例执行**：逐用例查看/更新状态（passed/failed/blocked），填写实际结果和备注
- **执行历史**：查看每个用例的执行历史记录
- **进度概览**：进度条 + 百分比

---

## 数据流向
```
TestPlan创建 → POST /api/runs/plans/ → 自动生成TestRun(每项目) → 返回计划+Run列表
用例执行 → PATCH /api/runs/run_cases/:id/update_status/ → 更新状态 → 写入历史 → 刷新进度统计
```
