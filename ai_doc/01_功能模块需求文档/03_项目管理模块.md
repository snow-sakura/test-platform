# 03 - 项目管理模块

## 模块定位
测试项目的基础管理单元，所有测试用例、版本、评审、执行计划等均归属到具体项目，是平台数据隔离和权限控制的核心维度。

---

## 后端需求

### 3.1 项目模型（Project）

| 字段 | 类型 | 说明 |
|------|------|------|
| name | CharField(200) | 项目名称 |
| description | TextField | 项目描述 |
| status | CharField(20) | active/archived/completed |
| start_date | DateField | 开始日期 |
| end_date | DateField | 结束日期 |
| members | M2M(User) | 项目成员 |
| created_by | FK(User) | 创建者 |
| created_at | DateTimeField | 创建时间 |
| updated_at | DateTimeField | 更新时间 |

### 3.2 API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/projects/` | GET | 项目列表（仅返回用户参与的项目） |
| `/api/projects/` | POST | 创建项目（自动添加创建者为成员） |
| `/api/projects/{id}/` | GET | 项目详情 |
| `/api/projects/{id}/` | PUT/PATCH | 更新项目信息 |
| `/api/projects/{id}/` | DELETE | 删除项目 |
| `/api/projects/{id}/members/` | POST | 添加项目成员 |
| `/api/projects/{id}/members/{user_id}/` | DELETE | 移除项目成员 |

### 3.3 权限控制
- **列表权限**：用户只能看到自己是成员的项目
- **详情权限**：用户必须是项目成员或超级管理员
- **写操作**：仅项目创建者和超级管理员可修改/删除
- **成员管理**：仅创建者可添加/移除成员

### 3.4 数据关联
- 项目是多个下游模块的外键：TestCase、TestSuite、Version、TestPlan、TestRun、ApiProject、UiProject、AppProject 等

---

## 前端需求

### 3.5 页面组件

| 页面 | 路由 | 功能 |
|------|------|------|
| ProjectList.vue | `/ai-generation/projects` | 项目列表，含搜索、状态筛选、CRUD 操作 |
| ProjectDetail.vue | `/ai-generation/projects/:id` | 项目详情页 |

### 3.6 列表页功能
- **搜索**：按项目名称模糊搜索
- **筛选**：按状态（active/archived/completed）筛选
- **表格列**：项目名称、描述、状态标签、成员数、创建时间、操作
- **操作**：新建项目（对话框表单）、编辑、删除、查看详情

### 3.7 新建/编辑表单
- 项目名称（必填）
- 项目描述（选填，支持多行文本）
- 状态选择（active/archived/completed）
- 开始日期/结束日期
- 成员管理（多选用户）

---

## 数据流向
```
ProjectList.vue → GET /api/projects/ → 后端按用户权限过滤 → 返回项目列表
ProjectForm → POST/PUT /api/projects/ → 校验权限 → 保存到 DB → 返回结果
```
