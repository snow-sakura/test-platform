# 07 - 评审管理模块

## 模块定位
测试用例评审的完整工作流，支持创建评审、分配评审人、提交评审意见、使用评审模板和检查清单。确保测试用例质量。

---

## 后端需求

### 7.1 评审模型（TestCaseReview）

| 字段 | 类型 | 说明 |
|------|------|------|
| title | CharField(500) | 评审标题 |
| description | TextField | 评审描述 |
| projects | M2M(Project) | 关联项目 |
| testcases | M2M(TestCase) | 评审的用例 |
| creator | FK(User) | 创建人 |
| reviewers | M2M(User) | 评审人员（通过 ReviewAssignment） |
| template | FK(ReviewTemplate) | 使用的评审模板 |
| status | CharField(20) | pending/in_progress/approved/rejected/cancelled |
| priority | CharField(20) | low/medium/high/urgent |
| deadline | DateTimeField | 截止日期 |
| created_at | DateTimeField | 创建时间 |
| updated_at | DateTimeField | 更新时间 |
| completed_at | DateTimeField | 完成时间 |

### 7.2 评审分配模型（ReviewAssignment）

| 字段 | 类型 | 说明 |
|------|------|------|
| review | FK(TestCaseReview) | 关联评审 |
| reviewer | FK(User) | 评审人 |
| status | CharField(20) | pending/approved/rejected/abstained |
| comment | TextField | 评审意见 |
| checklist_results | JSONField | 检查清单结果 |
| reviewed_at | DateTimeField | 评审时间 |
| assigned_at | DateTimeField | 分配时间 |
| **约束** | unique_together | review + reviewer |

### 7.3 评审意见模型（TestCaseReviewComment）

| 字段 | 类型 | 说明 |
|------|------|------|
| review | FK(TestCaseReview) | 关联评审 |
| testcase | FK(TestCase) | 关联用例（可选） |
| author | FK(User) | 评论者 |
| comment_type | CharField(20) | general/testcase/step |
| content | TextField | 意见内容 |
| step_number | PositiveIntegerField | 步骤序号 |
| is_resolved | BooleanField | 是否已解决 |
| created_at | DateTimeField | 创建时间 |
| updated_at | DateTimeField | 更新时间 |

### 7.4 评审模板模型（ReviewTemplate）

| 字段 | 类型 | 说明 |
|------|------|------|
| name | CharField(200) | 模板名称 |
| description | TextField | 模板描述 |
| project | M2M(Project) | 关联项目 |
| creator | FK(User) | 创建人 |
| checklist | JSONField | 检查清单（JSON数组） |
| default_reviewers | M2M(User) | 默认评审人 |
| is_active | BooleanField | 是否启用 |
| created_at | DateTimeField | 创建时间 |
| updated_at | DateTimeField | 更新时间 |

### 7.5 API 端点

**评审管理**：
| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/reviews/reviews/` | GET | 评审列表（支持项目/状态/评审人筛选） |
| `/api/reviews/reviews/` | POST | 创建评审 |
| `/api/reviews/reviews/{id}/` | GET/PUT/DELETE | 详情/更新/删除 |
| `/api/reviews/reviews/{id}/assign_reviewers/` | POST | 分配评审人员 |
| `/api/reviews/reviews/{id}/submit_review/` | POST | 提交评审结果（自动汇总状态） |
| `/api/reviews/reviews/my_reviews/` | GET | 我的评审任务 |

**评审意见**：
| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/reviews/review-comments/` | GET/POST | 意见列表/创建 |
| `/api/reviews/review-comments/{id}/` | GET/PUT/DELETE | 详情/更新/删除 |

**评审模板**：
| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/reviews/review-templates/` | GET/POST | 模板列表/创建 |
| `/api/reviews/review-templates/{id}/` | GET/PUT/DELETE | 详情/更新/删除 |

### 7.6 核心业务逻辑
- **评审流程**：创建评审 → 分配评审人 → 评审人提交意见 → 自动汇总结果
- **自动汇总**：全部评审人 approved → 评审状态自动变为 approved；任一 rejected → 自动变为 rejected
- **检查清单**：模板定义检查项，评审时逐项打勾
- **权限控制**：创建者可管理评审，评审人只能提交自己的意见

## 前端需求

### 7.7 页面组件

| 页面 | 路由 | 功能 |
|------|------|------|
| ReviewList.vue | `/ai-generation/reviews` | 评审列表，支持项目/状态/评审人筛选 |
| ReviewDetail.vue | `/ai-generation/reviews/:id` | 评审详情，含基本信息、进度、评审分配和意见 |
| ReviewForm.vue | `/ai-generation/reviews/create` | 新建/编辑评审（选择项目、优先级、截止日期、模板） |
| ReviewTemplateList.vue | `/ai-generation/review-templates` | 评审模板列表管理 |

### 7.8 列表页功能
- **筛选**：项目下拉、状态下拉（pending/in_progress/approved/rejected）、评审人筛选
- **表格列**：标题、关联项目、状态标签、优先级标签、评审人数、截止日期、操作
- **操作**：查看详情、编辑、删除

### 7.9 详情页功能
- **基本信息**：标题、描述、项目、优先级、截止日期、状态
- **评审进度**：各评审人状态（pending/approved/rejected/abstained）
- **评审意见**：按用例展示评审意见，支持筛选未解决的意见
- **操作**：分配评审人、提交评审结果（含检查清单打勾）

---

## 数据流向
```
ReviewList → GET /api/reviews/reviews/ → 按权限过滤 → 返回评审列表
ReviewForm → POST /api/reviews/reviews/ → 创建评审+分配评审人 → 返回详情
ReviewDetail → submit_review → 检查清单+意见 → 自动汇总状态
```
