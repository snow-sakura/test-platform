# 01 - 用户认证与权限模块

## 模块定位
平台基础模块，负责用户注册、登录、Token 管理和权限控制。所有其他模块依赖此模块的认证体系。

---

## 后端需求

### 1.1 用户模型
- **自定义 User 模型**：基于 Django AbstractUser 扩展
- **字段**：username、email、first_name、last_name、department、position、is_active、date_joined
- **JWT Token 机制**：
  - access_token 有效期 60 分钟
  - refresh_token 有效期 7 天
  - 刷新时旧 token 自动加入黑名单
  - 支持自动刷新检测（过期前 5 分钟提前刷新）

### 1.2 API 端点

| 端点 | 方法 | 功能 | 认证 |
|------|------|------|------|
| `/api/auth/login/` | POST | 用户登录，返回双 token 和用户信息 | 否 |
| `/api/auth/register/` | POST | 用户注册 | 否 |
| `/api/auth/logout/` | POST | 退出登录，refresh token 加入黑名单 | 是 |
| `/api/auth/token/refresh/` | POST | 刷新 access_token | 否（使用 refresh_token） |
| `/api/auth/profile/` | GET | 获取当前用户信息 | 是 |
| `/api/auth/profile/` | PUT/PATCH | 更新用户资料（基本信息、修改密码） | 是 |
| `/api/users/` | GET | 用户列表（供其他模块选择执行人/评审人） | 是 |

### 1.3 认证机制
- **默认认证类**：JWT → Token → Session 优先级链
- **默认权限类**：IsAuthenticated（所有 API 默认需登录）
- **CSRF 豁免**：自定义 DisableCSRFMiddleware 中间件
- **CORS 支持**：django-cors-headers，允许跨域携带 Cookie

### 1.4 权限控制
- **项目级权限**：用户只能访问自己参与的项目
- **基于项目的数据隔离**：testcases、executions、reviews 等均按项目权限过滤
- **超级管理员豁免**：is_superuser 可访问所有数据

## 前端需求

### 1.5 页面组件

| 页面 | 路由 | 功能 |
|------|------|------|
| Login.vue | `/login` | 登录表单（用户名/邮箱 + 密码），左侧品牌展示 |
| Register.vue | `/register` | 注册表单（用户名、邮箱、姓名、密码、确认密码） |
| UserProfile.vue | `/profile` | 个人资料（基本信息 Tab + 修改密码 Tab） |

### 1.6 状态管理（useUserStore）
- **状态**：user、accessToken、refreshToken、tokenExpiresAt
- **计算属性**：isAuthenticated、isTokenExpiringSoon（5 分钟内）、isTokenExpired
- **核心操作**：
  - `login()`：POST `/auth/login/`，保存双 token，启动自动刷新
  - `register()`：POST `/auth/register/`
  - `logout()`：POST `/auth/logout/`，清除本地状态，跳转登录页（防重复标记）
  - `refreshAccessToken()`：POST `/auth/token/refresh/`
  - `initAuth()`：应用初始化，恢复用户状态，检查 token 有效性

### 1.7 路由守卫
- `requiresAuth`：未登录跳转 `/login`
- `requiresGuest`：已登录跳转 `/home`
- `beforeEach`：有 token 但无 user 时自动调用 initAuth()

### 1.8 Axios 拦截器
- **请求拦截**：自动注入 `Bearer <access_token>`
- **响应拦截**：401 时自动尝试刷新 token，并发请求排队防止重复刷新
- **Token 预刷新**：即将过期时自动刷新

## 数据流向
```
用户操作 → Vue组件 → Pinia Store → Axios(带Token) → DRF认证 → 业务逻辑 → 响应
                                                                      ↓
                                                               JWT验证/刷新
```
