# 05 - 测试套件管理模块

## 模块定位
将测试用例按执行顺序组织为测试套件，支持在测试执行中批量运行。用于管理回归测试集、冒烟测试集等用例集合。

---

## 后端需求

### 5.1 测试套件模型（TestSuite）

| 字段 | 类型 | 说明 |
|------|------|------|
| project | FK(Project) | 关联项目 |
| name | CharField(200) | 套件名称 |
| description | TextField | 套件描述 |
| testcases | M2M(TestCase) | 关联用例（通过 TestSuiteCase） |
| author | FK(User) | 创建者 |
| created_at | DateTimeField | 创建时间 |
| updated_at | DateTimeField | 更新时间 |

### 5.2 套件用例关联模型（TestSuiteCase）

| 字段 | 类型 | 说明 |
|------|------|------|
| testsuite | FK(TestSuite) | 关联套件 |
| testcase | FK(TestCase) | 关联用例 |
| order | PositiveIntegerField | 执行顺序 |
| **约束** | unique_together | testsuite + testcase 联合唯一 |

### 5.3 API 端点
> **注意**：当前 testsuites app 的 urls.py 为空，API 端点待实现。模型已定义，可被其他模块引用。

| 端点（规划） | 方法 | 功能 |
|------|------|------|
| `/api/testsuites/` | GET | 套件列表 |
| `/api/testsuites/` | POST | 创建套件 |
| `/api/testsuites/{id}/` | GET/PUT/DELETE | 详情/更新/删除 |
| `/api/testsuites/{id}/add_case/` | POST | 添加用例到套件 |
| `/api/testsuites/{id}/remove_case/` | POST | 从套件移除用例 |
| `/api/testsuites/{id}/update_order/` | POST | 更新用例执行顺序 |

---

## 前端需求

### 5.4 页面组件

| 页面 | 路由 | 功能 |
|------|------|------|
| TestSuiteList.vue | `/ai-generation/testsuites` | 测试套件列表（当前为占位符） |

### 5.5 规划功能
- **套件列表**：项目筛选、搜索、CRUD
- **用例管理**：添加/移除用例、拖拽排序
- **批量运行**：一键执行套件内所有用例（依赖执行模块）
- **执行记录**：查看套件历史执行记录

---

## 数据流向
```
TestSuiteList → GET /api/testsuites/ → 返回套件列表
套件详情 → 包含 testcases 嵌套、order 排序
```
