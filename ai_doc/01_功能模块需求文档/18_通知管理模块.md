# 18 - 通知管理模块

## 模块定位
统一的即时通讯通知系统，支持飞书机器人、企业微信机器人、钉钉机器人的 Webhook 通知。可灵活配置通知渠道，与接口测试、UI 自动化、APP 自动化的定时任务联动。

---

## 后端需求

### 18.1 统一通知配置模型（UnifiedNotificationConfig）

| 字段 | 类型 | 说明 |
|------|------|------|
| name | CharField(100) | 配置名称 |
| config_type | CharField(20) | webhook_feishu / webhook_wechat / webhook_dingtalk |
| webhook_bots | JSONField | 机器人配置（dict，含 bot_key、secret 等） |
| is_default | BooleanField | 是否默认配置 |
| is_active | BooleanField | 是否启用 |
| created_by | FK(User) | 创建者 |
| created_at / updated_at | DateTimeField | 时间戳 |

### 18.2 API 端点（Core App 统一管理）

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/core/notification-configs/` | GET | 通知配置列表 |
| `/api/core/notification-configs/` | POST | 创建通知配置 |
| `/api/core/notification-configs/{id}/` | GET | 配置详情 |
| `/api/core/notification-configs/{id}/` | PUT/PATCH | 更新配置 |
| `/api/core/notification-configs/{id}/` | DELETE | 删除配置 |
| `/api/core/notification-configs/{id}/set_default/` | POST | 设置为默认配置 |
| `/api/core/notification-configs/active_configs/` | GET | 获取所有启用的配置 |

### 18.3 各模块通知体系

**接口测试通知**：
| 模型 | 端点前缀 | 功能 |
|------|----------|------|
| ApiNotificationConfig | `/api/api-testing/notification-configs/` | 通知配置 CRUD |
| ApiNotificationLog | `/api/api-testing/notification-logs/` | 通知日志 + 重试发送 |

**UI 自动化通知**：
| 模型 | 端点前缀 | 功能 |
|------|----------|------|
| UiNotificationLog | `/api/ui-automation/notification-logs/` | 通知日志 + retry |
| UiTaskNotificationSetting | `/api/ui-automation/task-notification-settings/` | 任务-通知关联设置 |

**APP 自动化通知**：
| 模型 | 端点前缀 | 功能 |
|------|----------|------|
| AppNotificationLog | `/api/app-automation/notification-logs/` | 通知日志 + retry |

### 18.4 核心业务逻辑
- **多机器人支持**：一个通知配置可包含多个 Webhook 机器人
- **渠道分类**：飞书、企业微信、钉钉独立配置
- **默认配置**：每个类型只能有一个默认配置
- **Webhook 推送**：调用对应平台 Webhook API 发送消息（Markdown/文本格式）
- **重试机制**：发送失败时支持手动 retry
- **任务关联**：定时任务可绑定通知配置，执行完成/失败时自动发送通知

| 页面 | 路由 | 功能 |

### 18.6 通知配置页功能
- **渠道选择**：飞书机器人 / 企业微信机器人 / 钉钉机器人
- **Webhook 配置**：Webhook URL、签名密钥
- **多机器人**：添加多个机器人，每个独立启用/禁用
- **默认设置**：设为默认配置
- **测试发送**：测试 Webhook 连接

### 18.7 通知日志功能
- **列表**：通知类型、接收渠道、状态（成功/失败/重试中）、发送时间
- **筛选**：按状态、时间范围筛选
- **重试**：失败的日志可手动重试发送
- **搜索**：按配置名称搜索

## 数据流向
```
通知配置 → POST /core/notification-configs/ → 保存 webhook_bots → set_default

定时任务完成 → 查找关联的 TaskNotificationSetting → 获取启用的通知配置
           → 调用 Webhook API → 发送消息 → 写入 NotificationLog

通知日志 retry → POST /notification-logs/:id/retry/ → 重新调用 Webhook → 更新状态
```
