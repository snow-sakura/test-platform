# 15 - AI 智能模式模块（Browser-Use）

## 模块定位
基于 browser-use 库 + LangChain 的 AI 驱动智能浏览器自动化。用户通过自然语言描述测试任务，AI 自动规划、执行浏览器操作并生成测试报告。

---

### 15.3 核心业务逻辑

**AI 执行流程**（`ai_agent.py` using browser-use + LangChain）：
```
1. 用户输入自然语言任务 →
2. 创建 AIExecutionRecord →
3. 后台线程执行 run_full_process_sync →
   a. AI 分析任务 → 规划步骤（planned_tasks）
   b. 使用 browser-use Agent + LangChain 模型执行浏览器操作
   c. 调用 Selenium/Playwright 执行实际浏览器交互
   d. 每步更新 step_update 回调 → 写入 execution_log
   e. 任务状态自动流转（backfill_prior_pending_tasks）
4. 完成后：
   a. resolve_execution_status → 汇总子任务状态 → 整体 pass/fail
   b. 处理 GIF 录制 → 保存到 media/ai_recording/
   c. append_execution_summary → 生成摘要
```

**取消机制**：
- 全局 STOP_SIGNALS 字典：`{execution_id: bool}`
- run 时检查 should_stop_async（实时）→ should_stop_sync（最终）
- stop_task 设置信号 + 直接更新 DB 状态

**故障分类**：
- **基础设施故障**（Execution LLM unavailable、connection errors、timeouts）
- **任务故障**（正常测试失败）

**AI 模型角色**：
- `browser_use_text`：文本模式推理（更快）
- `browser_use_vision`：视觉模式推理（需要截图分析）

**GIF 录制**：
- browser-use Agent 自动记录操作 GIF
- 执行完成后将 `agent_history.gif` 移动到 `media/ai_recording/` 并重命名

**报告生成**：
- AIExecutionReportGenerator：生成汇总/详细/性能报告
- AIReportPDFGenerator：导出 PDF 格式报告

## 前端需求

### 15.4 页面组件

| 页面 | 路由 | 功能 |
|------|------|------|
| AITesting.vue | `/ai-intelligent-mode/testing` | **AI 智能测试主页** |
| AICaseList.vue | `/ai-intelligent-mode/cases` | AI 用例列表管理 |
| AIExecutionRecords.vue | `/ai-intelligent-mode/executions` | AI 执行记录列表 |
| AIExecutionReport.vue | `/ai-intelligent-mode/report/:id` | AI 执行报告详情 |

### 15.5 AI 测试主页功能
- **任务描述输入**：多行文本，自然语言描述测试目标
- **目标 URL 输入**：要测试的网页地址
- **模式选择**：文本模式 / 视觉模式
- **选项**：是否录制 GIF
- **执行按钮** → 后台执行 → 实时日志展示 → 显示计划任务列表
- **停止按钮**：执行中可随时停止

### 15.6 执行记录功能
- **列表**：状态筛选、时间排序、分页
- **详情**：任务描述、执行日志、步骤完成情况、计划任务状态
- **操作**：查看报告、导出 PDF、批量删除
- **GIF 回放**：查看录制的操作 GIF

### 15.7 报告功能
- 汇总报告：总体通过/失败、步骤数、耗时
- 详细报告：每步操作的详细描述和结果
- 性能报告：各步骤响应时间
- PDF 导出：一键下载 PDF 格式报告

## 数据流向
```
AI测试主页 → 输入自然语言任务 → POST /ai-cases/:id/run/ → 后台线程：
  → browser-use Agent + LangChain (DeepSeek/Qwen等)
  → Selenium/Playwright 浏览器操作
  → 每步更新 execution_log + planned_tasks
  → 完成后汇总状态 + GIF录制

临时执行 → POST /ai-executions/run_adhoc/ → 同上流程
停止 → POST /ai-executions/:id/stop_task/ → STOP_SIGNALS[id]=True → 线程退出
报告 → GET /ai-executions/:id/export_pdf/ → 生成PDF → 下载
```
