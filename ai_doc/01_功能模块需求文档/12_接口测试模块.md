# 12 - 接口测试模块（API Testing）

## 模块定位
完整的 HTTP/WebSocket 接口测试平台，支持接口管理、环境管理、自动化测试套件、定时任务、请求历史和 Allure 报告。

---

## 后端需求

### 12.1 模型体系

**ApiProject（API 项目）**：名称、描述、类型、状态、成员、创建时间

**ApiCollection（接口集合）**：
| 字段 | 类型 | 说明 |
|------|------|------|
| project | FK(ApiProject) | 关联项目 |
| name | CharField(200) | 集合名称 |
| parent | FK(self) | 父集合（树形结构） |
| created_at | DateTimeField | 创建时间 |

**ApiRequest（接口请求）**：
| 字段 | 类型 | 说明 |
|------|------|------|
| collection | FK(ApiCollection) | 关联集合 |
| name | CharField(200) | 请求名称 |
| method | CharField(10) | GET/POST/PUT/DELETE/PATCH/HEAD/OPTIONS |
| url | URLField | 请求 URL |
| headers | JSONField | 请求头 |
| query_params | JSONField | URL 参数 |
| body | JSONField | 请求体 |
| expected_response | JSONField | 预期响应 |
| is_favorite | BooleanField | 是否收藏 |

**ApiTestSuite（API 测试套件）**：名称、项目、请求列表（JSON）、定时任务关联

**ApiEnvironment（环境）**：
| 字段 | 类型 | 说明 |
|------|------|------|
| name | CharField(200) | 环境名称 |
| env_type | CharField(20) | global/local |
| variables | JSONField | 环境变量 |
| is_active | BooleanField | 是否启用 |

**ApiRequestHistory（请求历史）**：
| 字段 | 类型 | 说明 |
|------|------|------|
| request | FK(ApiRequest) | 关联请求 |
| method | CharField(10) | 请求方法 |
| url | URLField | 请求 URL |
| headers | JSONField | 请求头 |
| body | JSONField | 请求体 |
| response_status | IntegerField | 响应状态码 |
| response_body | TextField | 响应体 |
| elapsed_time | FloatField | 响应时间（ms） |
| executed_at | DateTimeField | 执行时间 |

**ApiScheduledTask（定时任务）**：名称、类型、套件、cron 表达式、触发方式、状态、最后执行时间

**ApiNotificationConfig/Log（通知配置/日志）**：类似通用通知模块

### 12.2 API 端点

**仪表盘**：
| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/api-testing/dashboard/stats/` | GET | 项目数、接口数、套件数、今日执行数 |

**项目管理**：CRUD + 成员管理

**接口管理**：
| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/api-testing/collections/` | GET/POST | 集合列表/创建（树形结构） |
| `/api/api-testing/collections/{id}/` | GET/PUT/DELETE | 详情/更新/删除 |
| `/api/api-testing/requests/` | GET/POST | 请求列表/创建 |
| `/api/api-testing/requests/{id}/` | GET/PUT/DELETE | 详情/更新/删除 |
| `/api/api-testing/requests/{id}/execute/` | POST | **执行单个请求** |

**自动化测试**：
| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/api-testing/test-suites/` | GET/POST | 套件列表/创建 |
| `/api/api-testing/test-suites/{id}/` | GET/PUT/DELETE | 详情/更新/删除 |
| `/api/api-testing/test-suites/{id}/execute/` | POST | **执行整个套件** |

**环境管理**：CRUD + 激活

**请求历史**：列表/批量删除/搜索

**定时任务**：CRUD + 暂停/恢复/立即执行 + 执行日志

**通知管理**：通知配置 + 通知日志

**Allure 报告**：报告生成和查看

### 12.3 核心业务逻辑
- **请求执行**：根据请求配置拼接 URL/Headers/Body，使用 httpx 发送请求，记录历史
- **环境变量**：全局环境和本地环境，执行时注入环境变量
- **变量解析**：通过 core.variable_resolver 解析 `${function_name(args)}` 动态变量
- **定时执行**：基于 cron 表达式 + Celery 定时执行套件
- **WebSocket**：支持 WebSocket 连接测试

## 前端需求

### 12.4 页面组件（共 10 个页面 + 3 个组件）

| 页面 | 路由 | 功能 |
|------|------|------|
| index.vue | `/api-testing` | 模块布局，二级水平导航 |
| Dashboard.vue | `/api-testing/dashboard` | 统计仪表盘 |
| ProjectManagement.vue | `/api-testing/projects` | API 项目 CRUD |
| InterfaceManagement.vue | `/api-testing/interfaces` | 接口管理（左侧集合树 + 右侧详情） |
| AutomationTesting.vue | `/api-testing/automation` | 自动化测试套件 |
| RequestHistory.vue | `/api-testing/history` | 请求历史（HTTP/WebSocket 两Tab） |
| EnvironmentManagement.vue | `/api-testing/environments` | 环境管理（全局/本地两Tab） |
| ReportView.vue | `/api-testing/reports` | Allure 报告 |
| ScheduledTasks.vue | `/api-testing/scheduled-tasks` | 定时任务管理 |
| AIServiceConfig.vue | `/api-testing/ai-service` | AI 服务配置 |
| NotificationManagement.vue | `/api-testing/notifications` | 通知管理 |

### 12.5 接口管理核心功能
- **左侧集合树**：支持多级嵌套，拖拽排序，右键菜单（新增/重命名/删除）
- **右侧接口详情**：
  - 请求方法下拉（GET/POST/PUT/DELETE/PATCH/HEAD/OPTIONS）
  - URL 输入框（支持环境变量 `${base_url}`）
  - 请求头 KeyValueEditor
  - URL 参数 KeyValueEditor
  - Body 编辑器（JSON/FormData/FormUrlencoded），Monaco Editor
  - 预期响应定义
  - 发送按钮 + 响应区域（状态码、耗时、响应体格式化）

### 12.6 自动化测试功能
- **套件列表**：左侧项目选择 + 套件列表
- **套件详情**：套件内请求列表，支持添加/移除/排序
- **执行**：点击执行 → 实时显示每个请求的执行结果（通过/失败/耗时）
- **断言**：基于预期响应自动判断通过/失败

### 12.7 环境管理功能
- **全局环境**：全局共享的环境变量
- **本地环境**：项目级环境变量
- **变量表格**：Key-Value 表格编辑器
- **激活**：点击激活，当前环境高亮显示

---

## 数据流向
```
接口管理 → 选择集合树节点 → GET /requests/?collection_id= → 右侧编辑
执行请求 → POST /requests/:id/execute/ → httpx 发送 → 记录历史 → 显示响应
执行套件 → POST /test-suites/:id/execute/ → 遍历请求 → 逐条断言 → 汇总结果
定时任务 → Celery Beat → 触发套件执行 → 通知
```
