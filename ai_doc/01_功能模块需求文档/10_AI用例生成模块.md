# 10 - AI 用例生成模块（需求分析）

## 模块定位
AI 驱动的测试用例生成平台核心模块。支持上传需求文档（PDF/Word/TXT）、AI 自动分析需求、流式生成测试用例、AI 评审和修订，最终导入到测试用例库。

---

### 10.3 核心业务逻辑

**AI 用例生成全流程（异步线程）**：
```
1. 创建任务 → 验证 writer/reviewer 模型和提示词已激活
2. Stream模式：调用 AIModelService.generate_test_cases_stream
   → SSE 回调推送增量内容
   → 每500字符保存到 stream_buffer
3. 自动评审（如 GenerationConfig.auto_review=True）
   → 调用 review_test_cases_stream
   → 评审意见写入 review_feedback
4. 修订（根据评审意见）
   → 调用 revise_test_cases_based_on_review
   → review_timeout 超时控制
5. 排序、重新编号 → 保存到 final_test_cases
6. Complete模式：同上非流式版本
```

**SSE 流式推送机制**：
- 创建 StreamingHttpResponse（text/event-stream）
- 每 0.5s 轮询数据库，推送增量 buffer/review/final 内容
- 心跳每 10s 发送
- 最大超时 1 小时
- CORS 头处理

**用例内容解析**：
- `_parse_test_cases_content`：优先表格格式（含管道符处理），其次文本格式
- `_parse_table_format`：列头匹配 caseId/scenario/precondition/steps/expected/priority
- `_parse_text_format`：关键字检测提取字段

**用例保存到用例库**：
- 解析 final_test_cases 内容
- 创建 TestCase + TestCaseStep 记录
- 自动创建默认项目（如不存在）

## 前端需求

### 10.4 页面组件

| 页面 | 路由 | 功能 |
|------|------|------|
| RequirementAnalysisView.vue | `/ai-generation/requirement-analysis` | **主页**：需求输入、AI生成用例、配置引导弹窗 |
| GeneratedTestCaseList.vue | `/ai-generation/generated-cases` | AI 生成的任务列表，状态筛选、批量操作 |
| TaskDetail.vue | `/ai-generation/tasks/:id` | 单个任务详情 |
| AIModelConfig.vue | `/ai-generation/ai-model-config` | AI 模型配置 |
| PromptConfig.vue | `/ai-generation/prompt-config` | 提示词配置 |
| GenerationConfigView.vue | `/ai-generation/generation-config` | 生成行为配置 |

### 10.5 主页功能（RequirementAnalysisView）
- **配置引导弹窗**：检查 AI 模型和提示词是否已配置，未配置引导跳转
- **需求输入区**：
  - 文件上传（PDF/Word/TXT，拖拽上传）
  - 纯文本输入
  - 上传后自动提取文本
- **AI 分析区**：点击分析 → 显示分析进度 → 展示提取的需求列表
- **用例生成区**：
  - 选择需求 → 设置测试级别/优先级/数量
  - 点击生成 → 实时流式展示（SSE）
  - 显示评审反馈
  - 最终用例表格展示
- **用例操作**：批量导入到用例库、单条编辑、批量删除

### 10.6 配置页功能
- **AIModelConfig**：选择 AI 提供商（DeepSeek/Qwen/SiliconFlow/OpenAI），配置 API 地址/密钥/模型名/温度/Token 数，测试连接
- **PromptConfig**：管理 tester.md 和 tester_pro.md 提示词，加载默认、编辑、预览
- **GenerationConfig**：设置默认测试级别/优先级/数量/是否自动评审/评审超时，单例激活模式

### 10.7 任务列表功能
- **筛选**：状态筛选（pending/generating/reviewing/completed/failed）
- **表格列**：任务ID、状态标签、进度、模式、创建时间、操作
- **统计概览**：各状态任务数量统计

## 数据流向
```
需求文档上传 → POST /documents/ → 提取文本 → analyze → 生成 BusinessRequirement 列表
                                                    ↓
选择需求 + 配置 → POST /tasks/:id/generate/ → 异步线程：
  → writer_config + writer_prompt → AI 生成用例(流式/完整)
  → reviewer_config + reviewer_prompt → AI 评审
  → 修订 → final_test_cases

SSE前端 → GET /tasks/:id/stream_progress_sse/ → 每0.5s推送增量 → 实时渲染
保存 → POST /tasks/:id/save_to_records/ → 解析 → TestCase + TestCaseStep 入库
```
