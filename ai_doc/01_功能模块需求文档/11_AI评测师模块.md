# 11 - AI 评测师模块（智能助手）

## 模块定位
集成 Dify AI 平台的智能对话助手，支持会话管理、连续对话、消息持久化。可作为 AI 评测师辅助测试相关工作。

---

## 后端需求

### 11.1 模型体系

**DifyConfig（Dify 配置）**：
| 字段 | 类型 | 说明 |
|------|------|------|
| api_url | URLField(500) | Dify API 地址 |
| api_key | CharField(500) | API 密钥 |
| is_active | BooleanField | 是否启用 |
| created_at | DateTimeField | 创建时间 |
| updated_at | DateTimeField | 更新时间 |

**AssistantSession（会话）**：
| 字段 | 类型 | 说明 |
|------|------|------|
| user | FK(User) | 用户 |
| session_id | CharField(200) | 会话 ID |
| conversation_id | CharField(200) | Dify 对话 ID（保持连续性） |
| title | CharField(500) | 会话标题 |
| created_at / updated_at | DateTimeField | 时间戳 |

**ChatMessage（聊天消息）**：
| 字段 | 类型 | 说明 |
|------|------|------|
| session | FK(AssistantSession) | 关联会话 |
| role | CharField(20) | user/assistant |
| content | TextField | 消息内容 |
| conversation_id | CharField(200) | Dify 对话 ID |
| message_id | CharField(200) | Dify 消息 ID |
| created_at | DateTimeField | 创建时间 |

### 11.2 API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/assistant/sessions/` | GET/POST | 会话列表/创建 |
| `/api/assistant/sessions/{id}/` | GET/PUT/DELETE | 详情/更新/删除 |
| `/api/assistant/sessions/{id}/messages/` | GET | 获取会话消息列表 |
| `/api/assistant/chat/send_message/` | POST | **发送消息到 Dify**（SSE 流式响应） |
| `/api/assistant/config/dify/` | GET/POST | Dify 配置 CRUD |
| `/api/assistant/config/dify/test_connection/` | POST | 测试 Dify API 连接 |

### 11.3 核心业务逻辑
- **发送消息**：POST `/chat/send_message/`，将用户消息转发到 Dify API，流式返回 AI 回复
- **会话连续性**：通过 `conversation_id` 保持 Dify 对话上下文
- **消息持久化**：用户消息和 AI 回复均存入数据库
- **配置管理**：支持管理端配置 Dify API 地址和密钥，支持测试连接

## 前端需求

### 11.4 页面组件

| 页面 | 路由 | 功能 |
|------|------|------|
| AssistantView.vue | `/ai-generation/assistant` | AI 对话主界面 |

### 11.5 对话界面功能
- **左侧历史会话列表**：
  - 显示用户的所有历史会话
  - 支持新建会话、切换会话、删除会话
  - 会话按时间倒序排列
- **右侧聊天区域**：
  - 消息气泡（用户/AI 区分左右）
  - 流式打字效果（SSE）
  - 输入框 + 发送按钮
  - 支持 Enter 发送
- **对话连续性**：切换会话恢复历史消息

### 11.6 配置页（ConfigurationCenter 中）
- Dify API URL 输入
- API Key 输入（脱敏显示）
- 测试连接按钮
- 启用/禁用切换

---

## 数据流向
```
用户输入 → POST /api/assistant/chat/send_message/ → 后端转发到 Dify API
                                                        ↓
                                                SSE 流式返回 AI 回复
                                                        ↓
                                        用户消息和AI回复均存入 ChatMessage

AssistantView → GET /sessions/ → 会话列表
             → GET /sessions/:id/messages/ → 加载历史消息
```
