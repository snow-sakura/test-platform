# 06 - 版本管理模块

## 模块定位
管理项目版本/迭代，支持基线版本标记。版本是测试计划、测试执行、评审等的重要关联维度。

---

## 后端需求

### 6.1 版本模型（Version）

| 字段 | 类型 | 说明 |
|------|------|------|
| projects | M2M(Project) | 关联项目（多对多） |
| name | CharField(100) | 版本名称 |
| description | TextField | 版本描述 |
| is_baseline | BooleanField | 是否为基线版本 |
| created_by | FK(User) | 创建者 |
| created_at | DateTimeField | 创建时间 |
| updated_at | DateTimeField | 更新时间 |

### 6.2 API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/versions/` | GET | 版本列表（仅用户有权限的项目） |
| `/api/versions/` | POST | 创建版本（校验项目权限） |
| `/api/versions/{id}/` | GET | 版本详情 |
| `/api/versions/{id}/` | PUT/PATCH | 更新版本 |
| `/api/versions/{id}/` | DELETE | 删除版本 |
| `/api/versions/projects/{project_id}/versions/` | GET | 获取指定项目的版本列表 |

### 6.3 核心业务逻辑
- **权限过滤**：用户只能看到自己有权限的项目的版本
- **基线版本**：每个项目可标记一个版本为基线（作为测试基准）
- **多项目关联**：一个版本可关联多个项目

---

## 前端需求

### 6.4 页面组件

| 页面 | 路由 | 功能 |
|------|------|------|
| VersionList.vue | `/ai-generation/versions` | 版本列表 CRUD |

### 6.5 列表页功能
- **搜索**：按名称模糊搜索
- **筛选**：项目筛选、基线版本筛选
- **表格列**：名称、关联项目、描述、是否基线、创建者、创建时间、操作
- **操作**：新建版本（对话框表单）、编辑、删除、批量删除

---

## 数据流向
```
VersionList → GET /api/versions/?project_id= → 后端按权限过滤 → 返回版本列表
VersionForm → POST /api/versions/ → 校验项目权限 → 保存 → 返回结果
```
