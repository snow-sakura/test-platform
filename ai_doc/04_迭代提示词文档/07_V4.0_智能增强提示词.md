# V4.0 智能增强与配置中心 - 开发提示词

## 需求提点

实现高阶能力整合：AI 智能模式（browser-use 自然语言驱动浏览器）、配置中心统一入口（整合所有配置页面）、统一通知管理（多机器人 Webhook）、国际化完善、性能优化。

这是最后一个大版本，重点是整合和增强，而非新增独立模块。

关键技术点：browser-use Agent + LangChain、全局 STOP_SIGNALS 取消机制、GIF 录制、PDF 报告生成、Vue Router 嵌套路由 + fade 过渡。

---

## 执行计划

### 第一阶段：AI 智能模式（最核心）

**步骤1：实现 AI 用例和执行记录模型**
- AICase（project, name, task_description TextField, target_url, execution_mode text/vision, enable_gif, planned_tasks JSON, status）
- AIExecutionRecord（ai_case FK nullable, project, task_description, execution_mode, enable_gif, steps_completed, planned_tasks JSON, execution_log TextField, gif_recording FileField, status, started_at, completed_at）

**步骤2：实现 browser-use Agent 集成**
- ai_agent.py：run_full_process_sync 函数
- browser-use Agent + LangChain ChatOpenAI（使用 AIModelConfig 的 browser_use_text/vision 角色）
- 执行流程：
  - AI 分析任务 → 规划步骤 → 存入 planned_tasks
  - browser-use Agent 执行浏览器操作（Playwright）
  - 每步 on_step_update 回调 → 更新 execution_log
  - 任务状态自动流转（backfill_prior_pending_tasks）
  - 完成后 resolve_execution_status + summarize_planned_tasks
- 数据库连接安全管理（MySQL gone-away 重试）
- GJANGO_ALLOW_ASYNC_UNSAFE 设置

**步骤3：实现取消机制**
- 全局 STOP_SIGNALS 字典：`{execution_id: bool}`
- should_stop_async（实时检查）+ should_stop_sync（最终检查）
- stop_task：设置 STOP_SIGNALS + 直接更新 DB 状态

**步骤4：实现 GIF 录制**
- browser-use 自动录制 agent_history.gif
- _process_gif_recording：移动到 media/ai_recording/ + 时间戳命名

**步骤5：实现故障分类**
- is_infrastructure_failure：检查错误消息关键词
- 区分基础设施故障（LLM 不可用/连接错误/超时）和任务故障

**步骤6：实现 AI 执行 API**
- AICaseViewSet：CRUD + **run 接口**（后台线程执行）
- AIExecutionRecordViewSet：
  - CRUD + batch_delete
  - **run_adhoc 接口**（不绑定用例的临时执行）
  - **stop_task 接口**
  - generate_report：AIExecutionReportGenerator
  - export_pdf：AIReportPDFGenerator → HttpResponse application/pdf

**步骤7：实现 AI 智能模式前端**
- AITesting.vue：自然语言输入 + URL + 模式选择 + GIF开关 + 执行/停止 + 实时日志
- AICaseList.vue：AI 用例 CRUD + 执行
- AIExecutionRecords.vue：执行记录列表/详情/批量删除
- AIExecutionReport.vue：汇总/详细/性能报告 + PDF 导出 + GIF 回放

### 第二阶段：配置中心统一

**步骤8：实现配置中心容器**
- ConfigurationCenter.vue：
  - 侧边栏菜单（AI模型/提示词/生成配置/UI环境/APP环境/智能模式/Dify/通知）
  - router-view + fade 过渡动画
- 复用已有配置子页面组件
- 统一路由结构 /configuration/*

**步骤9：实现配置状态概览**
- 检查各配置项的完成状态
- 引导用户完成未配置的项

### 第三阶段：统一通知管理

**步骤10：实现统一通知配置**
- UnifiedNotificationConfig（name, config_type webhook_feishu/webhook_wechat/webhook_dingtalk, webhook_bots JSON, is_default, is_active）
- CRUD + set_default + active_configs
- 多机器人支持（一个配置多个 Webhook 机器人）

**步骤11：实现通知配置前端**
- NotificationConfigs.vue：渠道选择 + Webhook 配置 + 多机器人列表 + 测试发送

### 第四阶段：国际化完善和性能优化

**步骤12：国际化完善**
- 补全 18 个语言文件的所有翻译条目
- 统一中英文术语
- 日期/时间/数字格式化
- Element Plus 国际化全面覆盖

**步骤13：性能优化**
- 后端：select_related/prefetch_related 查询优化、适当缓存、大文件分页
- 前端：路由懒加载、组件异步加载、Tree Shaking、静态资源优化

**步骤14：首页增强**
- 导航卡片增加状态指示（哪些模块已配置可用）
- 完善快速入口
