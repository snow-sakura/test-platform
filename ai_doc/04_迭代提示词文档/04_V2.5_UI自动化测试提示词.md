# V2.5 UI 自动化测试 - 开发提示词

## 需求提点

实现 Web UI 自动化测试平台，采用 Page Object 模式。核心链路：元素管理（定位器+分组+验证+AI建议）→ 页面对象（元素关联+代码生成）→ 脚本编辑（步骤编排+执行）→ 测试用例/套件 → 执行追踪（截图+操作记录）→ 定时任务+通知。

关键技术点：Selenium/Playwright 浏览器自动化、多类型定位器（id/name/css/xpath等）、备用定位器自动切换、Page Object 代码自动生成、Celery Beat 定时调度。

---

## 执行计划

### 第一阶段：元素管理（核心）

**步骤1：实现元素模型体系**
- UiProject（name, description, url, browser_type, status, members）
- LocatorStrategy（name, type, description, priority）
- Element（project, name, locator_type, locator_value, backup_locators JSON, group FK, page_url）
- ElementGroup（name, project, parent FK, order，树形结构）

**步骤2：实现元素 API**
- 元素 CRUD + 分组管理
- **定位器验证接口**：使用 Selenium 在浏览器中实时查找 → 返回找到/未找到 + 匹配数量
- 备用定位器 CRUD（add_backup_locator 等）
- **AI 定位建议接口**：AI 分析页面 HTML → 生成候选定位器列表
- 元素使用情况查询（usages）
- 元素树结构（tree）

**步骤3：实现元素管理前端**
- ElementList.vue（基础版）
- ElementManagerEnhanced.vue（增强版）
- 左侧分组树：多级拖拽、右键菜单
- 中间元素列表：按分组筛选
- 右侧详情/编辑：定位器类型选择 + 值输入 + 备用定位器列表 + 验证按钮 + AI建议按钮

### 第二阶段：页面对象和脚本

**步骤4：实现页面对象模型**
- PageObject（project, name, url, elements M2M, generated_code TextField）
- PageObjectElement（page_object, element, alias, order）

**步骤5：实现页面对象 API**
- CRUD + 元素关联（add_element, elements 列表）
- **生成代码接口**：根据元素列表生成 Page Object 类代码

**步骤6：实现脚本模型**
- TestScript（project, name, page_object FK）
- ScriptStep（script, step_number, action_type, element FK, input_value, expected_result）
- ScriptElementUsage（script, element, usage_count, context）

**步骤7：实现脚本 API**
- CRUD + 步骤 CRUD + 批量创建步骤
- 脚本元素使用分析（analyze_script）

**步骤8：实现脚本前端**
- ScriptList.vue
- ScriptEditorEnhanced.vue：
  - 左侧步骤列表（拖拽排序）
  - 右侧步骤编辑（操作类型/元素选择/输入值/预期结果）
  - 批量添加（从 Page Object）
  - 代码视图（Monaco Editor）
  - 运行测试

### 第三阶段：测试用例、套件、执行

**步骤9：实现 UI 测试用例**
- UiTestCase（project, name, script FK, priority, status, test_data JSON）
- CRUD + copy_case + run + batch_run

**步骤10：实现 UI 套件和执行**
- UiTestSuite（name, project, testcases M2M+order）
- 套件 CRUD + 用例管理 + **run_suite 接口**
- TestExecution（suite, status, result, 时间线）
- Screenshot（execution, step, image）
- OperationRecord（execution, step, action_type, detail）
- 执行 CRUD + run + abort

**步骤11：实现执行前端**
- TestCaseManager.vue
- SuiteList.vue：用例/脚本管理 + 执行
- ExecutionList.vue：执行记录 + 截图 + 操作记录

### 第四阶段：定时任务、通知、环境、仪表盘

**步骤12：定时任务和通知**
- UiScheduledTask + UiNotificationLog + UiTaskNotificationSetting
- CRUD + pause/resume/run_now + 通知日志 retry

**步骤13：环境配置**
- UiEnvironment（browser_type, window_size, timeout, screenshot settings）
- CRUD

**步骤14：前端剩余页面**
- ScheduledTasks.vue + NotificationConfigs.vue + NotificationLogs.vue
- Dashboard.vue
- ProjectList.vue + PageObjectManager.vue

## 实现流程

### 后端流程
```
1. ui_automation app → UiProject/LocatorStrategy/Element/ElementGroup 模型
2. 元素 API（含定位器验证/备用定位器/AI建议）
3. PageObject/PageObjectElement 模型 → Page Object API + 代码生成
4. TestScript/ScriptStep/ScriptElementUsage 模型 → 脚本 API
5. UiTestCase/UiTestSuite/TestExecution 模型 → 执行 API
6. Screenshot/OperationRecord 模型 → 截图/记录 API
7. UiScheduledTask 模型 → Celery 集成
8. UiNotificationLog → Webhook 推送
```

### 前端流程
```
1. ElementManagerEnhanced.vue（最复杂，三栏布局+分组树+定位器验证）
2. PageObjectManager.vue
3. ScriptEditorEnhanced.vue（步骤编排+代码视图）
4. TestCaseManager.vue + SuiteList.vue
5. ExecutionList.vue
6. ScheduledTasks.vue + 通知页面
7. Dashboard.vue
```
