# V1.5 测试用例与评审体系 - 开发提示词

## 需求提点

在 V1.0 基础上，建立完整的测试用例全生命周期管理体系：测试用例管理（含步骤/附件/评论/Excel导入导出）、测试套件管理、版本管理、评审管理（含模板/分配/检查清单/自动汇总）、执行管理（计划→TestRun→逐用例执行→历史追踪）、报告管理（Dashboard/图表/趋势/AI效能）。

核心逻辑链路：创建用例 → 加入套件 → 关联版本 → 提交评审 → 通过后加入执行计划 → 逐用例执行 → 生成报告。

---

## 执行计划

### 第一阶段：测试用例管理

**步骤1：实现 TestCase 四表模型**
- TestCase（project, title, description, preconditions, priority, status, type, author）
- TestCaseStep（testcase, step_number, action, expected_result）
- TestCaseAttachment（testcase, file, filename, file_size, uploaded_by）
- TestCaseComment（testcase, author, content）

**步骤2：实现用例 API**
- 用例 CRUD（嵌套步骤序列化）
- 步骤 CRUD（添加/更新/删除/排序）
- 附件上传/删除
- 评论 CRUD
- 列表筛选（项目/优先级/状态/类型）+ 搜索 + 分页
- Excel 导出（含步骤，openpyxl）
- Excel 批量导入（解析 → 批量创建用例+步骤）
- 批量删除

**步骤3：实现用例前端页面**
- TestCaseList.vue：搜索/筛选/批量操作/导出按钮
- TestCaseForm.vue：基本信息表单 + 动态步骤行 + 附件上传
- TestCaseDetail.vue：Tab 切换（信息/步骤/附件/评论）
- TestCaseEdit.vue：编辑页面

### 第二阶段：测试套件与版本

**步骤4：实现套件模型和 API**
- TestSuite（project, name, description, testcases, author）
- TestSuiteCase（testsuite, testcase, order，唯一约束）
- 套件 CRUD + 用例关联管理（添加/移除/排序）

**步骤5：实现版本模型和 API**
- Version（projects M2M, name, description, is_baseline, created_by）
- 版本 CRUD（按项目权限过滤）
- 按项目获取版本列表

**步骤6：实现套件和版本前端**
- TestSuiteList.vue
- VersionList.vue：项目筛选/基线筛选/批量删除

### 第三阶段：评审管理

**步骤7：实现评审模型体系**
- TestCaseReview（title, description, projects, testcases, creator, reviewers, template, status, priority, deadline）
- ReviewAssignment（review, reviewer, status, comment, checklist_results，唯一约束）
- TestCaseReviewComment（review, testcase, author, comment_type, content, step_number, is_resolved）
- ReviewTemplate（name, description, checklist JSON, default_reviewers, is_active）

**步骤8：实现评审 API**
- 评审 CRUD + assign_reviewers + submit_review（自动汇总：全部 approved→approved，任一 rejected→rejected）
- 评审意见 CRUD
- 评审模板 CRUD
- my_reviews（我的评审任务）

**步骤9：实现评审前端页面**
- ReviewList.vue：项目/状态/评审人筛选
- ReviewDetail.vue：基本信息 + 评审进度 + 评审意见列表
- ReviewForm.vue：选择项目/优先级/截止日期/模板 + 分配评审人
- ReviewTemplateList.vue：模板管理 + 检查清单编辑

### 第四阶段：执行管理

**步骤10：实现执行模型体系**
- TestPlan（name, description, projects, version, creator, assignees, is_active）
- TestRun（name, test_plan, project, version, testcases, assignee, status, 时间线）
- TestRunCase（test_run, testcase, status, priority, actual_result, comments, defects JSON, elapsed_time，唯一约束）
- TestRunCaseHistory（run_case, status, actual_result, executed_by, executed_at）

**步骤11：实现执行 API**
- 计划 CRUD（创建时 perform_create 自动为每个项目生成 TestRun）
- 执行 CRUD + 用例关联
- 用例执行状态更新（PATCH update_status → 自动写入 History）
- 执行历史查看
- 进度统计（动态计算 total/passed/failed/blocked/untested）

**步骤12：实现执行前端页面**
- ExecutionListView.vue：计划列表 + 状态筛选
- ExecutionDetailView.vue：TestRun 列表 + 统计卡片 + 进度条 + 逐用例执行

### 第五阶段：报告管理

**步骤13：实现报告模型和 API**
- TestReport（project, name, report_type, execution, summary JSON, content JSON）
- ReportTemplate（name, template_config JSON, is_default）
- Dashboard 概览接口
- 执行状态分布接口
- 缺陷分布接口
- 失败用例 TOP10 接口
- 每日执行趋势接口（默认 7 天）
- AI 效能分析接口
- 团队工作量统计接口

**步骤14：实现报告前端页面**
- AiTestReport.vue：项目/时间筛选 + 统计卡片 + ECharts 图表
- 执行状态分布饼图
- 缺陷分布柱状图
- 失败用例 TOP10 横向柱状图
- 每日执行趋势折线图
- AI 效能对比图
- 团队工作量柱状图

## 实现流程

### 后端流程
```
1. testcases app → 四表模型 → CRUD API → Excel导入导出
2. testsuites app → 两表模型 → CRUD API
3. versions app → 一表模型 → CRUD API
4. reviews app → 四表模型 → CRUD API（含评审流程）
5. executions app → 四表模型 → CRUD API（含计划→Run自动创建）
6. reports app → 两表模型 → Dashboard聚合API
```

### 前端流程
```
1. TestCaseList → TestCaseForm → TestCaseDetail → TestCaseEdit
2. TestSuiteList
3. VersionList
4. ReviewList → ReviewForm → ReviewDetail → ReviewTemplateList
5. ExecutionListView → ExecutionDetailView
6. AiTestReport（含ECharts图表）
```
