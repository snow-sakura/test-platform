# V2.0 接口测试 - 开发提示词

## 需求提点

实现完整的 HTTP/WebSocket 接口测试平台。核心能力：接口管理（多级集合树 + 请求编辑 + 发送执行）、自动化测试套件（批量执行 + 断言）、环境管理（全局/本地变量）、请求历史、Celery 定时任务、Allure 报告、通知系统。

关键技术点：httpx 发送请求、变量解析器 `${function_name()}`、Celery Beat cron 调度。

---

## 执行计划

### 第一阶段：API 项目和接口管理

**步骤1：实现模型体系**
- ApiProject（name, description, type, status, members）
- ApiCollection（project, name, parent FK自引用，树形结构）
- ApiRequest（collection, name, method, url, headers JSON, query_params JSON, body JSON, expected_response JSON, is_favorite）

**步骤2：实现集合和请求 API**
- 集合 CRUD（树形，支持多级嵌套）
- 请求 CRUD
- **单请求执行接口**：httpx 发送 → 记录 ApiRequestHistory → 返回响应

**步骤3：实现接口管理前端**
- InterfaceManagement.vue
- 左侧 Element Plus Tree：多级嵌套、右键菜单（新增/重命名/删除）
- 右侧请求编辑区：
  - 方法下拉（GET/POST/PUT/DELETE/PATCH/HEAD/OPTIONS）
  - URL 输入（支持 `${base_url}`）
  - 请求头 KeyValueEditor
  - URL 参数 KeyValueEditor
  - Body 编辑器（JSON/FormData/FormUrlencoded，Monaco Editor）
  - 预期响应定义
  - 发送按钮
- 响应区：状态码、耗时、响应体 JSON 格式化

### 第二阶段：自动化测试

**步骤4：实现套件模型和 API**
- ApiTestSuite（name, project, requests JSON, 定时任务关联）
- 套件 CRUD + 请求列表管理
- **套件执行接口**：遍历请求 → httpx 发送 → 断言比较 → 汇总通过/失败

**步骤5：实现自动化测试前端**
- AutomationTesting.vue
- 左侧：项目选择 + 套件列表
- 右侧：套件内请求列表（添加/移除/排序）
- 执行按钮 + 实时结果展示

### 第三阶段：环境和历史

**步骤6：实现环境管理**
- ApiEnvironment（name, env_type global/local, variables JSON, is_active）
- 环境 CRUD + 激活
- 执行时注入环境变量替换

**步骤7：实现环境管理前端**
- EnvironmentManagement.vue
- 全局/本地两个 Tab
- 变量 Key-Value 表格编辑
- 激活按钮

**步骤8：实现请求历史**
- ApiRequestHistory（request FK, method, url, headers, body, response_status, response_body, elapsed_time, executed_at）
- 历史列表（搜索/筛选/分页）+ 批量删除

**步骤9：实现请求历史前端**
- RequestHistory.vue
- HTTP / WebSocket 两个 Tab
- 搜索 + 批量删除

### 第四阶段：定时任务

**步骤10：实现 Celery 定时任务**
- ApiScheduledTask（name, type, suite FK, cron, 触发方式, status, last_run）
- Celery Beat 配置和 schedule 动态更新
- CRUD + pause/resume/run_now
- 执行日志记录

**步骤11：实现定时任务前端**
- ScheduledTasks.vue
- 类型/触发方式/状态筛选
- Cron 表达式配置 + 验证
- 暂停/恢复/立即执行

### 第五阶段：报告、通知、仪表盘

**步骤12：实现报告**
- Allure 报告生成和查看

**步骤13：实现通知系统**
- ApiNotificationConfig（webhook 配置）
- ApiNotificationLog（发送日志 + status + retry_count）
- 配置 CRUD + 测试发送
- 日志 + retry

**步骤14：实现通知前端**
- NotificationManagement.vue：配置 Tab + 日志 Tab

**步骤15：实现仪表盘**
- Dashboard 统计接口
- Dashboard.vue：统计卡片

## 实现流程

### 后端流程
```
1. api_testing app → ApiProject/ApiCollection/ApiRequest 模型 → CRUD API
2. 请求执行逻辑（httpx + 历史记录）
3. ApiTestSuite 模型 → 套件执行逻辑（遍历+断言）
4. ApiEnvironment 模型 → 变量注入
5. ApiRequestHistory 模型 → 历史查询
6. Celery Beat + ApiScheduledTask 模型 → 定时调度
7. Allure 集成
8. ApiNotificationConfig/Log 模型 → Webhook发送
9. Dashboard 聚合统计
```

### 前端流程
```
1. InterfaceManagement.vue（最复杂，集合树+请求编辑+发送+响应）
2. KeyValueEditor 公共组件
3. AutomationTesting.vue
4. EnvironmentManagement.vue + EnvironmentTable
5. RequestHistory.vue + HistoryTable
6. ScheduledTasks.vue
7. ReportView.vue
8. NotificationManagement.vue
9. Dashboard.vue
```
