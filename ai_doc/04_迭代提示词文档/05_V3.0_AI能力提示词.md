# V3.0 AI 能力 - 开发提示词

## 需求提点

引入 AI 驱动测试能力，包含三大子模块：

1. **AI 用例生成**（需求分析 → AI 生成 → 评审 → 导入用例库）：实现需求文档上传/文本提取、AI 模型/提示词/生成行为配置、异步生成任务（Stream + Complete 模式）、SSE 流式推送、生成结果解析和保存。

2. **AI 评测师**（Dify 集成）：会话管理、消息持久化、Dify API 流式转发。

3. **数据工厂**（80+ 测试数据工具）：7 大分类工具集、批量生成、变量函数解析器（`${function_name()}`）。

关键技术点：OpenAI 兼容 SDK 调用多 AI 提供商、Django StreamingHttpResponse SSE、Python threading 异步执行、PyPDF2/python-docx 文档处理。

## 执行计划

### 第一阶段：AI 配置体系（基础设施）

**步骤1：实现 AIModelConfig 模型和 API**
- AIModelConfig（name, model_type deepseek/qwen/siliconflow/openai, role, api_base, api_key, model_name, temperature, max_tokens, is_active）
- CRUD + enable/disable
- **test_connection**：异步调用 OpenAI 兼容 API（60s 超时）

**步骤2：实现 PromptConfig 模型和 API**
- PromptConfig（name, prompt_type writer/reviewer, content TextField, is_active）
- CRUD + enable/disable
- **load_defaults**：从 docs/tester.md、docs/tester_pro.md 加载

**步骤3：实现 GenerationConfig 模型和 API**
- GenerationConfig（name, test_level, test_priority, test_case_count, auto_review, review_timeout, is_active）
- CRUD + enable（单例激活模式）

### 第二阶段：需求分析和 AI 用例生成

**步骤4：实现需求分析模型**
- RequirementDocument（title, file, content, file_type, uploader）
- RequirementAnalysis（document, status, result JSON）
- BusinessRequirement（analysis, title, description, priority, category）

**步骤5：实现需求分析 API**
- 文档上传 + 文本提取（PyPDF2/python-docx）
- analyze 接口（AI 分析 → 生成 BusinessRequirement 列表）
- analyze_text（纯文本分析）
- upload_and_analyze（上传+分析一步完成）

**步骤6：实现 AI 用例生成核心模型**
- GeneratedTestCase（requirement, title, scenario, preconditions, steps, expected_result, priority, status）
- TestCaseGenerationTask（task_id unique, status, mode stream/complete, writer_config FK, reviewer_config FK, writer_prompt FK, reviewer_prompt FK, generation_config FK, generated_content, review_feedback, final_test_cases, stream_buffer, progress 0-100, is_saved_to_records）

**步骤7：实现生成任务核心 API（最重要）**
- 任务 CRUD
- **generate 接口**（后台线程执行全流程）：
  - Stream 模式：generate_test_cases_stream → SSE 回调 → auto_review → revise → final
  - Complete 模式：generate → review → revise → final
- **stream_progress_sse 接口**：StreamingHttpResponse, text/event-stream, 0.5s 轮询, 10s 心跳, 1h 超时
- progress 查询
- cancel 取消
- **save_to_records**：解析 final_test_cases → 创建 TestCase + TestCaseStep
- batch_adopt / batch_discard / discard_selected_cases / discard_single_case
- update_test_cases / statistics

**步骤8：实现用例解析器**
- _parse_test_cases_content：优先表格格式，其次文本格式
- _parse_table_format：列头匹配（caseId/scenario/precondition/steps/expected/priority）
- _parse_text_format：关键字检测提取字段
- _reconstruct_test_cases_content（保持原格式）

**步骤9：实现 AI 用例生成前端**
- RequirementAnalysisView.vue（主页）：配置引导弹窗 → 文件上传/文本输入 → 分析 → SSE 实时展示 → 结果操作
- GeneratedTestCaseList.vue（任务列表）：状态筛选/统计/批量操作
- TaskDetail.vue（任务详情）
- AIModelConfig.vue + PromptConfig.vue + GenerationConfigView.vue

### 第三阶段：AI 评测师（Dify）

**步骤10：实现 Dify 集成后端**
- DifyConfig（api_url, api_key, is_active）
- AssistantSession（user, session_id, conversation_id, title）
- ChatMessage（session, role, content, conversation_id, message_id）
- 会话 CRUD + 消息列表
- **send_message**：POST → httpx 转发到 Dify API → SSE 流式返回
- Dify 配置 CRUD + test_connection

**步骤11：实现 AI 评测师前端**
- AssistantView.vue：左侧会话列表 + 右侧聊天区域（消息气泡 + 流式打字）
- DifyConfig.vue：配置管理

### 第四阶段：数据工厂

**步骤12：实现数据工厂后端**
- DataFactoryRecord（user, tool_name, tool_category, input_data JSON, output_data JSON, tags JSON）
- 7 大分类 80+ 工具实现：
  - test_data_tools（11个）：中文姓名/手机号/邮箱/地址/身份证/公司名/银行卡/香港身份证/营业执照/经纬度/用户档案
  - json_tools（10个）：格式化/校验/转XML/转YAML/JSONPath/扁平化/对比/路径列表/CSV互转/Mock
  - string_tools（9个）：去空格/替换/转义/字数统计/文本对比/正则/大小写
  - encoding_tools（14个）：条形码/二维码/时间戳/进制/Unicode/Base64/URL/JWT
  - random_tools（11个）：整数/浮点/字符串/UUID/MAC/IP/日期/布尔/颜色/密码/序列
  - encryption_tools（9个）：MD5/SHA1/SHA256/SHA512/AES/哈希比对/密码强度/盐值
  - crontab_tools（4个）：生成/解析/下次执行/验证
- 辅助工具：银行卡 Luhn 算法、统一信用代码生成、图片工具
- API：categories、execute、batch_generate、history、statistics、tags、variable_functions

**步骤13：实现变量解析器**
- core/variable_resolver.py：`resolve_variables()` 全局单例
- 支持 `${function_name(args)}` 语法
- 集成所有 7 大分类工具函数

**步骤14：实现数据工厂前端**
- DataFactory.vue：分类/场景双视图切换 + 工具执行 + 批量生成 + 历史记录
- DataFactorySelector 组件（在接口测试中弹窗选择数据）

## 实现流程

### 后端流程
```
1. AIModelConfig → PromptConfig → GenerationConfig（配置体系）
2. RequirementDocument → RequirementAnalysis → BusinessRequirement（需求分析）
3. TestCaseGenerationTask（核心）+ SSE 流式推送 + 用例解析器
4. GeneratedTestCase（AI生成的用例记录）
5. DifyConfig → AssistantSession → ChatMessage（Dify集成）
6. DataFactoryRecord + 7大工具模块 + variable_resolver
```

### 前端流程
```
1. AIModelConfig.vue + PromptConfig.vue + GenerationConfigView.vue
2. RequirementAnalysisView.vue（最复杂，上传+分析+SSE+结果操作）
3. GeneratedTestCaseList.vue + TaskDetail.vue
4. AssistantView.vue（聊天界面）
5. DataFactory.vue + DataFactorySelector组件
```
