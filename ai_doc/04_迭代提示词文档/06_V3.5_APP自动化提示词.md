# V3.5 APP 自动化 - 开发提示词

## 需求提点

实现 Android APP 自动化测试，基于 Airtest 框架。核心链路：设备管理（ADB 发现/连接/截图）→ 元素管理（图像捕获/坐标/区域/文本）→ 包名管理 → 场景编排器（可视化拖拽）→ 测试用例/套件 → 执行追踪（WebSocket 实时推送）→ 定时任务+通知。

关键技术点：ADB 设备通信、Airtest 图像识别匹配、Canvas 框选元素、WebSocket 实时状态推送。

---

## 执行计划

### 第一阶段：设备和元素管理（核心差异化）

**步骤1：实现设备管理后端**
- AppProject（name, description, package_name, status, members）
- Device（device_id, device_name, platform, status available/locked/online/offline, connected_at）
- ADB 设备发现接口（扫描连接设备）
- 设备连接/断开/锁定/解锁
- **设备截图接口**（ADB screencap → 返回图像，15s 超时）

**步骤2：实现设备管理前端**
- DeviceList.vue：设备列表 + 连接/断开/锁定/解锁/截图/发现按钮
- 设备截图弹窗

**步骤3：实现元素管理后端**
- AppElement（project, name, element_type image/coordinate/region/text, image ImageField, coordinates JSON, threshold, image_category FK）
- AppImageCategory（name, project, order）
- 元素 CRUD + 图片上传
- 图片分类 CRUD

**步骤4：实现元素管理前端**
- ElementList.vue
- **CaptureElementDialog.vue（关键组件）**：
  - 获取设备截图 → Canvas 展示
  - 鼠标框选区域 → 提取坐标/区域/截图
  - 保存为 AppElement（image/coordinate/region）
- ManualElementDialog.vue：手动输入坐标/区域/文本
- 图片分类管理

### 第二阶段：包名和场景编排

**步骤5：实现包名管理**
- AppPackage（project, app_name, package_name, version_name）
- CRUD 前后端

**步骤6：实现场景编排器后端**
- AppTestCase（project, name, description, scene_data JSON, status, priority）
- scene_data 格式：步骤数组 [{action_type, element_id, input_value, wait_time, ...}]
- CRUD + executeTestCase（Airtest 执行）

**步骤7：实现场景编排器前端（核心）**
- **SceneBuilder.vue**：
  - 可视化编排画布
  - 拖拽操作步骤（点击/滑动/输入/等待/断言/截图）
  - 步骤参数配置面板
  - 从元素库选择元素
  - 预览执行

### 第三阶段：套件、执行、组件库

**步骤8：实现套件和执行**
- AppTestSuite（project, name, testcases M2M+order）
- AppTestExecution（suite, status pending/running/completed/error/stopped, result, 时间线）
- 套件 CRUD + 用例管理 + runTestSuite
- **WebSocket 实时状态推送**（Django Channels）

**步骤9：实现执行前端**
- TestCaseList.vue
- SuiteList.vue：用例管理 + 执行
- ExecutionList.vue：WebSocket 状态实时展示 + 停止按钮

**步骤10：实现组件库**
- AppComponentLibrary（project, name, component_type basic/custom, element_data JSON）
- 基础组件列表 + 自定义组件 CRUD + 导入导出

### 第四阶段：定时任务、通知、环境

**步骤11：定时任务和通知**
- AppScheduledTask + AppNotificationLog
- CRUD + pause/resume/run_now + 通知日志 retry

**步骤12：环境配置**
- AppConfig（project, adb_path, device_timeout, screenshot）
- CRUD + 前端 AppSettings.vue

**步骤13：仪表盘和模块首页**
- Dashboard.vue + Index.vue（功能卡片导航）
- ScheduledTasks.vue + NotificationLogs.vue

## 实现流程

### 后端流程
```
1. app_automation app → AppProject/Device 模型 → ADB 集成
2. AppElement/AppImageCategory 模型 → 元素管理 API
3. AppPackage 模型 → 包名 CRUD
4. AppTestCase 模型 → 场景编排 + executeTestCase（Airtest）
5. AppTestSuite/AppTestExecution 模型 → 套件+执行+WebSocket
6. AppComponentLibrary 模型 → 组件库
7. AppScheduledTask → Celery 集成
8. AppNotificationLog → Webhook 推送
```

### 前端流程
```
1. DeviceList.vue（ADB设备管理）
2. CaptureElementDialog.vue（Canvas框选元素，最关键组件）
3. ElementList.vue + ManualElementDialog.vue
4. PackageList.vue
5. SceneBuilder.vue（可视化编排器，最复杂）
6. TestCaseList.vue + SuiteList.vue
7. ExecutionList.vue（WebSocket实时状态）
8. ScheduledTasks.vue + NotificationLogs.vue
```
