# TestPlate 迭代版本需求规划

## 版本规划总览

### 迭代逻辑

TestPlate 的版本迭代遵循 **"基础先行、逐步增强"** 的核心理念：
1. 先搭建基础平台（用户/项目/首页）
2. 再建立核心测试资产（用例/套件/评审/执行）
3. 再实现自动化能力（接口测试 → UI 自动化）
4. 再引入 AI 能力（AI 用例生成/评测师/数据工厂）
5. 扩展到 APP 自动化
6. 最后实现智能增强（AI 代理/配置中心整合）

### 迭代依赖链

```
V1.0 基础平台 ──→ V1.5 测试用例与评审体系 ──→ V2.0 接口测试 ──→ V2.5 UI自动化测试
                                                      │                    │
                                                      └──────→ V3.0 AI能力 ←──┘
                                                                     │
                                                              V3.5 APP自动化
                                                                     │
                                                              V4.0 智能增强
```

### 版本总览表

| 版本 | 名称 | 业务重点 | 核心交付 |
|------|------|----------|----------|
| **V1.0** | 基础平台 | 平台骨架搭建 | 用户认证、项目管理、首页门户 |
| **V1.5** | 测试用例与评审体系 | 手工测试管理 | 用例管理、套件、版本、评审、执行、报告 |
| **V2.0** | 接口测试 | API 自动化 | HTTP/WebSocket 测试、环境管理、定时任务 |
| **V2.5** | UI 自动化测试 | Web UI 自动化 | 元素/页面对象/脚本/用例/套件/执行 |
| **V3.0** | AI 能力 | AI 驱动测试 | AI 用例生成、AI 评测师、数据工厂 |
| **V3.5** | APP 自动化 | 移动端测试 | Android 设备管理、元素捕获、场景编排 |
| **V4.0** | 智能增强 | 高阶能力整合 | AI 智能模式、配置中心统一、国际化完善 |

### 为什么这样排序？

1. **V1.0 必须在最前面**：用户认证是所有功能的前置依赖，没有登录就无法使用任何功能。项目管理是数据组织的核心维度。

2. **V1.5 在 V2.0 之前**：手工测试用例是自动化测试的基础——自动化测试执行的也是测试用例，需要用例库作为数据源。评审流程保证用例质量。

3. **V2.0 在 V2.5 之前**：接口测试比 UI 自动化更底层、更稳定、更快速，企业通常先做接口自动化再做 UI 自动化。

4. **V3.0 在 V2.5 之后**：AI 能力是增值服务，需要在基础测试体系完善后再引入。AI 生成的用例需要导入到用例库（依赖 V1.5）。

5. **V3.5 在 V3.0 之后**：APP 自动化是最复杂的自动化测试类型，技术栈独立（Airtest），放在接口和 Web 自动化之后。

6. **V4.0 在最后**：AI 智能模式（browser-use）是实验性高阶功能。配置中心是对前面所有配置的统一整合入口。

## 实际完成状态

由于技术栈迁移（Django → FastAPI，Vue → Next.js），版本迭代计划已根据实际项目进度调整：

| 版本 | 实际内容 | 状态 |
|------|----------|------|
| **V1.0** | 基础平台（用户认证 + 首页门户 + 项目管理） | ✅ 已完成 |
| **V1.5** | 文档管理模块（多格式上传/解析/OCR） | ✅ 已完成 |
| **V2.0** | AI 测试管理（测试点提取/测试用例生成/异步任务/Excel导出） | ✅ 已完成 |
| **V3.0** | 知识库 & 系统设置（ChromaDB RAG/LLM热更新/飞书通知） | ✅ 已完成 |
| **V3.5** | 接口测试模块（API项目/集合树/请求执行/环境变量/定时任务） | ✅ 已完成 |
| **V4.0** | UI 自动化测试模块（页面对象/元素/脚本/套件/执行） | ✅ 已完成 |
| **V4.5** | APP 自动化测试模块（设备管理/元素捕获/场景编排） | ✅ 已完成 |
| **V5.0** | AI 智能模式 + 数据工厂 + AI 评测师 + AI 用例生成 | ✅ 已完成 |
| **V6.0** | 平台增强与运维能力（RBAC/国际化/Docker/CI-CD） | 📋 规划中 |
