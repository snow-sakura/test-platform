# 需求文档 - AI 测试用例协作助手

## 1. 项目概述

### 1.1 项目定位
基于 FastAPI + React + AutoGen 0.7.5 的 AI 测试用例协作助手，通过**多智能体团队协作**实现测试用例的自动生成、评审和优化。

### 1.2 核心价值
- 将 AI 从"单兵问答"升级为"团队协作"，模拟真实团队的测试设计评审流程
- 降低测试用例设计门槛，提升测试覆盖率和质量
- 实时流式输出，用户可见每个智能体的思考过程

### 1.3 目标用户
- 软件测试工程师：快速生成高质量的测试用例
- 测试经理：评审团队输出的测试用例质量
- 开发工程师：自测阶段快速生成测试用例

## 2. 功能需求

### 2.1 多智能体测试用例协作（核心功能）

| 功能编号 | F-001 |
|---------|-------|
| **名称** | 多智能体测试用例协作生成 |
| **优先级** | P0 |
| **描述** | 用户输入测试需求后，三个 AI 智能体按"设计→评审→确认"流程协作，输出完整的测试用例集 |
| **验收标准** | 1. 用户输入需求后触发三智能体轮询协作<br>2. 每个智能体按角色输出对应的内容<br>3. 用户实时看到流式输出 |
| **输入** | 用户文本消息（如"设计登录功能的测试用例"） |
| **输出** | 测试用例集（表格格式） + 评审报告 + 最终确认 |

**智能体角色说明：**

| 智能体 | 角色 | 职责 |
|--------|------|------|
| 测试用例设计师（primary） | 高级测试架构师 | 运用等价类划分、边界值分析、判定表等方法生成结构化测试用例 |
| 测试用例评审师（critic） | 评审专家 | 从完整性、准确性、可执行性、覆盖率、优先级、一致性6个维度评审 |
| 用户评审（user） | 用户代理 | 模拟用户决策：接受/拒绝/要求修改 |

### 2.2 流式输出

| 功能编号 | F-002 |
|---------|-------|
| **名称** | SSE 实时流式输出 |
| **优先级** | P0 |
| **描述** | 采用 Server-Sent Events 协议，将 AI 的生成过程实时推送给前端，实现打字机效果 |
| **验收标准** | 1. 前端逐字显示 AI 输出<br>2. 智能体切换时有明确标记<br>3. 流式完成后自动结束 |

### 2.3 多智能体角色可视化

| 功能编号 | F-003 |
|---------|-------|
| **名称** | 多智能体角色可视化区分 |
| **优先级** | P1 |
| **描述** | 前端对不同智能体的输出使用不同颜色标识，并显示当前正在输出的智能体 |
| **验收标准** | 1. 测试用例设计师 → 蓝色<br>2. 测试用例评审师 → 橙色<br>3. 用户评审 → 绿色<br>4. 显示"正在输出: XXX"状态标签 |

### 2.4 消息历史

| 功能编号 | F-004 |
|---------|-------|
| **名称** | 对话消息管理 |
| **优先级** | P1 |
| **描述** | 管理用户和 AI 的对话历史，支持滚动查看 |
| **验收标准** | 1. 用户消息和 AI 回复按时间顺序展示<br>2. 支持自动滚动到最新消息<br>3. 消息动态动画显示 |

### 2.5 空状态引导

| 功能编号 | F-005 |
|---------|-------|
| **名称** | 空状态引导 |
| **优先级** | P2 |
| **描述** | 首次打开页面时显示引导文字和快捷建议 |
| **验收标准** | 1. 显示"欢迎来到 AI 测试用例协作助手"<br>2. 提供快捷建议（如"设计登录功能的测试用例"） |

### 2.6 多行输入

| 功能编号 | F-006 |
|---------|-------|

## 4. 数据需求

### 4.1 输入数据格式

```json
{
  "message": "设计一个登录功能的测试用例",
  "conversation_id": "default"
}
```

### 4.2 输出数据格式（SSE）

```
data: {"content": "逐字符输出内容"}
data: {"content": "\\n\\n【测试用例设计师】\\n\\n"}
data: {"content": "\\n\\n--- 用户评审完成 ---"}
data: [DONE]
```

### 4.3 环境变量

```env
# 模型配置
AUTOGEN_MODEL=Qwen/Qwen2-VL-72B-Instruct
API_BASE_URL=https://api.siliconflow.cn/v1
OPENAI_API_KEY=sk-xxx
AUTOGEN_TEMPERATURE=0.7
AUTOGEN_MAX_TOKENS=2000

# 服务配置
HOST=0.0.0.0
PORT=8000
```

## 5. 约束与假设

### 5.1 技术约束
- 后端必须使用 Python 3.9+ 和 FastAPI
- 前端必须使用 React 18 + Vite 5
- AI 框架必须使用 AutoGen 0.7.5+
- 流式传输必须使用 SSE 协议
- 模型 API 须兼容 OpenAI 格式

### 5.2 业务假设
- 用户能够清晰地描述测试需求
- AI 模型能够生成中文测试用例
- 网络连接能够正常访问 SiliconFlow API
