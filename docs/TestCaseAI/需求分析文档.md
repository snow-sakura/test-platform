# 智能测试用例生成系统 —— 需求分析文档

## 1. 产品概述

### 1.1 产品定位
智能测试用例生成系统是一款基于 AI 大模型的测试辅助工具。用户只需上传需求图片（思维导图、流程图、界面原型截图等），系统即可自动分析图片内容并生成结构化的专业测试用例，支持一键导出为格式化 Excel 文件。

### 1.2 目标用户
- 软件测试工程师
- 测试开发工程师
- 质量保证（QA）人员
- 产品经理/需求分析人员（用于快速生成验收测试用例）

### 1.3 产品愿景
通过 AI 大模型的多模态理解和测试设计能力，将测试用例设计的耗时从小时级降低到分钟级，提升测试团队的工作效率和用例覆盖率。

---

## 2. 用户需求分析

### 2.1 核心痛点
| 痛点 | 说明 |
|------|------|
| 测试用例设计耗时 | 需求评审后，人工编写用例耗时较长 |
| 覆盖率难以保证 | 人工编写容易遗漏边界条件和异常场景 |
| 格式不统一 | 不同测试人员的用例格式和粒度不一致 |
| 需求理解偏差 | 从需求文档到用例的过程中信息丢失 |
| 图片类需求难处理 | 思维导图、界面原型等非结构化需求难以直接转化为用例 |

### 2.2 用户场景

#### 场景一：需求评审后的用例生成
> 测试工程师参加完需求评审，手上有产品经理给的界面原型图和需求思维导图。上传图片，AI 自动分析并生成完整的测试用例。

#### 场景二：快速回归用例补充
> 开发新功能后，测试人员需要补充回归用例。上传界面截图，AI 自动识别新增功能点并生成对应的测试用例。

#### 场景三：用例评审与优化
> 高级测试工程师使用多 Agent 模式，先由用例设计 Agent 生成用例，再由评审 Agent 进行评审。

## 3. 功能需求

### 3.1 功能模块总览

```
智能测试用例生成系统
├── 图片分析模块
│   ├── 图片上传
│   ├── 图片格式校验
│   ├── 图片大小校验
│   ├── 多模态AI分析
│   └── 流式分析结果展示
├── 测试用例生成模块
│   ├── 基于分析结果的用例生成
│   ├── 流式生成过程展示
│   ├── 结构化测试用例展示
│   └── 用例详情查看
├── 一键生成模块
│   ├── 图片分析 + 用例生成串联
│   ├── 阶段状态提示
│   └── 中断控制
├── Excel 导出模块
│   ├── 测试用例导出为 Excel
│   ├── 下载用例模板
│   └── Excel 格式美化
└── 用户配置模块
    ├── 项目背景信息输入
    └── 特定需求要求输入
```

### 3.2 功能详情

#### FR-01：图片上传与预览
| 项目 | 说明 |
|------|------|
| **描述** | 用户可上传图片文件，支持拖拽或点击选择 |
| **输入** | 图片文件（JPEG/PNG/GIF/WebP/BMP） |
| **限制** | 文件类型限于图片格式，大小不超过 10MB |
| **输出** | 图片预览缩略图，自动进入下一步 |
| **异常处理** | 非图片格式提示、超限提示 |

#### FR-02：多模态图片分析
| 项目 | 说明 |
|------|------|
| **描述** | AI 分析上传的图片内容，提取测试需求 |
| **输入** | 图片数据 + 上下文背景 + 用户要求 |
| **模型** | Qwen/Qwen2-VL-72B-Instruct（支持视觉理解） |
| **输出** | SSE 流式输出分析结果，包含功能模块、业务流程、测试点等 |
| **分析能力** | 思维导图识别、流程图分析、界面元素识别 |

#### FR-03：测试用例自动生成
| 项目 | 说明 |
|------|------|
| **描述** | 基于分析结果，AI 自动生成结构化测试用例 |
| **输入** | 图片分析结果（结构化文本） + 用户补充信息 |
| **模型** | Qwen/Qwen2.5-72B-Instruct（文本生成） |
| **输出** | SSE 流式输出测试用例 JSON 数组 |
| **用例字段** | 编号、标题、模块、前置条件、步骤、预期结果、优先级、类型 |
| **生成要求** | 覆盖功能/性能/安全/兼容性，包含正向和反向测试，至少 20 条 |

#### FR-04：一键生成流程
| 项目 | 说明 |
|------|------|
| **描述** | 图片分析 + 用例生成串联执行，用户只需上传和点击 |
| **阶段** | 分析中 → 分析完成 → 生成中 → 完成 |
| **状态提示** | 实时显示当前阶段和进度 |
| **中断控制** | 支持用户手动停止生成过程 |

#### FR-05：测试用例展示与查看
| 项目 | 说明 |
|------|------|
| **描述** | 以表格形式展示生成的测试用例，支持查看详情 |
| **功能** | 分页、列排序、用例详情弹窗 |
| **详情显示** | 完整展示用例所有字段，步骤以有序列表呈现 |

#### FR-06：Excel 导出
| 项目 | 说明 |
|------|------|
| **描述** | 将测试用例导出为格式化的 Excel 文件 |
| **格式** | 表头蓝色背景白色文字、优先级颜色编码（P0红/P1橙/P2黄/P3绿）、自动换行、冻结首行 |
| **功能** | 导出当前用例集、下载空白模板 |

#### FR-07：用户配置
| 项目 | 说明 |
|------|------|
| **描述** | 用户可输入项目背景和特定需求，辅助 AI 更精准地生成用例 |
| **字段** | 项目背景（文本域）、具体要求（文本域） |

## 4. 非功能需求

### 4.1 性能需求
| 需求 | 指标 |
|------|------|
| 图片分析响应 | 首 token 返回 < 5 秒（基于模型响应） |
| 用例生成响应 | 流式输出，首条用例 < 10 秒 |
| 图片大小限制 | ≤ 10MB |
| 同时处理数 | 单用户（当前版本） |

### 4.2 可用性需求
| 需求 | 说明 |
|------|------|
| 操作引导 | 步骤条引导用户操作 |
| 状态反馈 | 处理中实时展示进度和内容 |
| 错误提示 | 友好错误提示，不暴露技术细节 |
| 交互反馈 | 按钮加载状态、禁用状态 |

### 4.3 兼容性需求
| 项目 | 说明 |
|------|------|
| 浏览器 | Chrome、Firefox、Safari、Edge |
| 前端框架 | React 18，现代浏览器 |
| AI 接口 | OpenAI API 兼容接口，可替换为其他服务商 |

### 4.4 可维护性需求
| 项目 | 说明 |
|------|------|
| 模型可配置 | 通过环境变量配置视觉模型和文本模型 |
| API 地址可配置 | 通过环境变量配置 AI 服务地址 |
| 代码结构清晰 | 前后端分离，职责明确 |

---

## 5. 约束与限制

| 约束 | 说明 |
|------|------|
| 技术栈 | 后端 Python FastAPI，前端 React + Ant Design |
| AI 依赖 | 需外部大模型 API 服务（默认 SiliconFlow） |
| 网络要求 | 前端需能访问后端 API |
| 数据安全 | .env 文件中的 API Key 需妥善保管 |
| 当前局限 | 单用户模式，无用户认证和权限管理 |
