### 1.2 数据流

```
用户输入 → InputArea → useSSE.sendMessage()
    → HTTP POST /api/chat
        → SSEHandler.send_message()
            → _process_conversation() [asyncio.create_task]
                → team.run_stream(user_message)
                    → AutoGen 智能体轮询输出
                → 收集消息 + 按 source 分组
                → _send_to_session() → event_queue.put()
    → SSE /api/stream/{session_id}
        → stream_events() [AsyncGenerator]
            → event_queue.get() 循环
            → _format_sse_event() → yield SSE 格式字符串
    → EventSource 接收事件
        → useSSE.js 的事件监听
            → handleAgentResponse() → messages.value.push()
    → MessageBubble 渲染
```

## 三、后端核心模块详解

### 3.1 `main.py` - 应用入口

**路由表**：

| 路径 | 方法 | 功能 | 请求/响应 |
|------|------|------|-----------|
| `/health` | GET | 健康检查 | → `{"status": "healthy"}` |
| `/api/session` | POST | 创建新会话 | → `{"session_id": "uuid"}` |
| `/api/chat` | POST | 发送消息 | ← `ChatRequest` → `{"status": "message_sent"}` |
| `/api/upload` | POST | 上传文件 | ← `UploadFile` → `{"filename": "...", "content": "..."}` |
| `/api/stream/{session_id}` | GET | SSE 流式响应 | → `StreamingResponse` (text/event-stream) |

**关键细节**：
- CORS 配置为 `allow_origins=["*"]`，生产环境需限制
- `ChatRequest` 的 `attachment` 字段为可选，用于携带附件文本
- 使用 `uvicorn.run()` 直接启动，`reload=True` 便于开发

### 3.2 `sse_handler.py` - SSE 处理器（核心）

这是整个系统最复杂的模块，承担会话管理和 AI 驱动职责。

**类：`SSEHandler`**

**属性**：
```python
sessions: Dict[str, Dict] = {
    session_id: {
        "queue": asyncio.Queue,        # 用户输入队列（用于等待用户输入场景）
        "team": RoundRobinGroupChat,   # AutoGen 团队实例
        "active": bool,                # 会话是否活跃
        "waiting_for_input": bool,     # 是否正在等待用户输入
        "event_queue": asyncio.Queue,  # SSE 事件队列（连接时创建，断开时删除）
        "task": asyncio.Task           # 当前运行的任务（可选）
    }
}
```

**关键方法**：

1. `create_session()` - 创建会话
   - 生成 UUID session_id
   - 创建 `asyncio.Queue` 作为用户输入队列
   - 创建 `user_input_func` 闭包（AutoGen 需要的用户输入回调）
   - 调用 `team_manager.create_team(user_input_func)` 创建团队
   - 初始化会话字典并返回 session_id

2. `_process_conversation()` - 处理对话（核心）
   - 接收 session_id、用户内容和附件
   - 将附件拼接到用户消息前（"附件内容：\n...\n\n用户需求：\n..."）
   - 创建 `TextMessage` 并调用 `team.run_stream()`
   - 在 `async for message in stream` 中收集所有智能体输出
   - 过滤 `source == "user_proxy"` 的消息
   - 按 source 分组收集，同一智能体合并输出
   - 通过 `_send_to_session()` 逐个推送 AgentResponse
   - 收到 `TaskResult` 时保存状态和历史

3. `stream_events()` - SSE 事件流（异步生成器）
   - 为会话创建 `event_queue`
   - 发送 `connected` 事件
   - 循环从 `event_queue.get()` 读取并 yield SSE 格式数据
   - 60 秒超时发送 keep-alive
   - 客户端断开时捕获 `CancelledError` 并清理

4. `send_message()` - 接收消息
   - 校验 session 存在且活跃
   - 如果正在等待用户输入，消息放入输入队列
   - 如果已有任务在运行，拒绝新消息
   - 检查是否有活跃 SSE 流（`event_queue` 存在）
   - 启动后台任务 `_process_conversation()`
   - 注册回调捕获异常并记录日志

**全局单例**：
```python
sse_handler = SSEHandler()
```

### 3.3 `team_manager.py` - AutoGen 团队管理

**类：`TeamManager`**

**智能体配置**：

| 智能体名 | name | system_message 要点 |
|---------|------|-------------------|
| 流程协调员 | coordinator | 管理流程、分配任务、总结、发送 TERMINATE |
| 测试用例编写员 | test_writer | 编写 Markdown 表格格式测试用例，覆盖正常/异常/边界 |
| 测试用例评审员 | test_reviewer | 评审完整性/准确性/可执行性，标记 REVIEW_REJECT/PASS |

**团队配置**：
```python
team = RoundRobinGroupChat(
    participants=[coordinator, test_writer, test_reviewer],
    termination_condition=TextMentionTermination("TERMINATE") | MaxMessageTermination(30),
)
```

**模型客户端**：
```python
model_client = OpenAIChatCompletionClient(
    model=config.DEEPSEEK_MODEL,       # 默认 deepseek-chat
    api_key=config.DEEPSEEK_API_KEY,
    base_url=config.DEEPSEEK_BASE_URL,  # 默认 https://api.deepseek.com/v1
    model_info={
        "vision": False,
        "function_calling": True,
        "json_output": True,
        "family": "unknown",
        "structured_output": False,
    },
)
```

**状态持久化**：
- `save_team_state()`：调用 `team.save_state()` 后写入 `team_state.json`
- `save_history()`：序列化消息列表写入 `team_history.json`
- `_load_team_state()`：启动时尝试从文件恢复团队状态
- 注意：状态文件以当前工作目录为基准（非绝对路径）

**输入函数闭包**（在 SSEHandler 中创建）：
```python
async def user_input_func(prompt: str, cancellation_token=None) -> str:
    # 发送 UserInputRequest 事件通知前端 AI 在等待输入
    # 从队列获取用户输入
    # 超时或异常时返回 "TERMINATE"
```

## 四、前端核心模块详解

### 4.1 `useSSE.js` - SSE 通信层（核心）

这是前端最关键的模块，封装了所有与后端的通信逻辑。

**暴露的状态**：
| 状态 | 类型 | 说明 |
|------|------|------|
| `messages` | `ref([])` | 消息列表 |
| `isConnected` | `ref(false)` | SSE 连接状态 |
| `isProcessing` | `ref(false)` | AI 是否正在处理 |
| `isWaitingForInput` | `ref(false)` | AI 是否在等待用户输入 |
| `sessionId` | `ref(null)` | 当前会话 ID |

**暴露的方法**：

| 方法 | 功能 | 说明 |
|------|------|------|
| `connect()` | 连接 SSE | 先创建 session，再建立 EventSource 连接，返回 Promise |
| `disconnect()` | 断开连接 | 关闭 EventSource，清除定时器 |
| `sendMessage({content, attachment})` | 发送消息 | POST /api/chat，返回 boolean |
| `clearMessages()` | 清空消息 | 重置 messages 数组 |

**SSE 事件监听映射**：

| 事件名 | 处理逻辑 |
|--------|---------|
| `connected` | 确认连接，日志记录 |
| `message`（通用） | 处理 UserInputRequest（标记等待输入）和 error |
| `processing` | isProcessing = true，isWaitingForInput = false |
| `AgentResponse` | 解析 JSON → `handleAgentResponse()` → 添加到消息列表 |
| `done` | isProcessing = false，isWaitingForInput = false |
| `error` | 显示错误消息，重置处理状态 |
| `onerror`（连接错误） | 自动重连机制（最多 5 次，间隔 3 秒） |

**重连机制**：
```javascript
// 触发条件：EventSource 的 readyState === CLOSED
// 重连间隔：3 秒
// 最大重试：5 次
// 成功后重置尝试计数
// 超过最大次数后提示用户"连接断开，请刷新页面重试"
// 注意：readyState === CONNECTING 时由浏览器自动重连，无需介入
```

**API 基础 URL**：
- 开发环境（`import.meta.env.DEV`）：`http://localhost:8000`
- 生产环境：空字符串（同源）

### 4.2 `ChatWindow.vue` - 聊天主窗口

**组件层级**：
```
AgentStatus          → 显示连接状态和 AI 处理状态
messages-container   → 可滚动消息列表
  ├─ empty-state     → 无消息时的空状态提示
  ├─ MessageBubble   → 消息气泡（v-for 循环）
  └─ typing-indicator → AI 思考中的打字动画
InputArea            → 输入框 + 文件上传
```

**关键逻辑**：
- 组件挂载时自动调用 `connect()` 
- 使用 `watch` 监听消息数量变化实现自动滚动到底部
- `TransitionGroup` 实现消息滑入动画

### 4.3 `MessageBubble.vue` - 消息气泡

**智能体配置表**：
```javascript
const agentConfig = {
  'test_writer':    { label: '测试用例编写员', icon: '✍️', color: '#52c41a' },
  'test_reviewer':  { label: '测试用例评审员', icon: '🔍', color: '#fa8c16' },
  'coordinator':    { label: '流程协调员',    icon: '🎯', color: '#722ed1' },
  'assistant':      { label: 'AI助手',        icon: '🤖', color: '#1677ff' },
}
```

**视觉特性**：
- 用户消息：右侧对齐，蓝色渐变背景，白色文字
- AI 消息：左侧对齐，左侧时间轴线和彩色圆点
- 智能体标签：左侧 3px 彩色边框 + 彩色圆形图标 + 名称
- 长内容（>500 字符）：底部"展开全部"按钮，折叠高度 150px

### 4.4 `InputArea.vue` - 输入区域

**组件功能**：
- 文本输入框：绑定 `inputValue`，Enter 键发送
- 附件上传：`a-upload` 组件，限制 `.pdf,.txt` 格式
- 附件预览：显示文件名，可关闭移除
- 发送按钮：loading 状态跟随 `isProcessing`

**文件上传流程**：
1. `handleFileUpload(file)` → `parseFile(file)` 解析文件
2. 成功 → 设置 `attachedFile` 和 `attachedContent`
3. 失败 → `message.error()` 显示错误
4. 发送后自动清除附件

**发送条件**（`canSend` computed）：
- `inputValue.trim()` 非空
- `!props.disabled`
- `!props.isProcessing`

### 4.5 `fileParser.js` - 前端文件解析

**技术选型**：pdfjs-dist（浏览器端 PDF 解析）

**API**：
```javascript
parseFile(file)      → 自动识别类型分发
parsePDF(file)       → ArrayBuffer → pdfjs.getDocument → 逐页提取文本
parseTextFile(file)  → file.text() 直接读取
```

## 七、开发与调试指南

### 7.1 本地开发环境搭建

```bash
# 后端
cd backend
cp .env.example .env
# 编辑 .env 填入 DEEPSEEK_API_KEY
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 前端
cd frontend
npm install
npm run dev    # 默认 localhost:3000，自动代理 /api → localhost:8000
```

### 7.2 Vite 代理配置

```javascript
// vite.config.js
server: {
  port: 3000,
  proxy: {
    '/api': {
      target: 'http://localhost:8000',
      changeOrigin: true
    }
  }
}
```

注意：`useSSE.js` 中的 `getBaseUrl()` 函数在 `import.meta.env.DEV` 为 true 时返回 `http://localhost:8000`，会绕过 Vite 代理直接请求后端。如需通过代理，需修改此逻辑。

### 7.3 调试技巧

**后端日志**（SSEHandler 中大量使用 `print()` 调试）：
- 查看会话创建和销毁
- 跟踪消息流转：`[DEBUG] Sending message to session xxx`
- 查看文件附件大小：`[DEBUG] Attached content (xxx chars) to message`
- 查看智能体输出：`[DEBUG] Message from source: test_writer`

**前端日志**：
- SSE 连接状态：`[DEBUG] Connecting to SSE: ...`
- 事件接收：`[DEBUG] Processing event received:`
- Agent 响应：`[DEBUG] AgentResponse from test_writer`

### 7.4 常见开发问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| SSE 连接失败 | EventSource 请求被浏览器 CORS 拦截 | 检查后端 CORS 配置 |
| 前端收到 connected 但收不到后续事件 | session 未正确创建 | 检查创建 session 后在 SSR 连接时是否传入了正确的 session_id |
| 文件上传后 AI 未识别内容 | attachment 字段未正确传递 | 检查 `_process_conversation()` 中的拼接逻辑 |
| 智能体输出混乱 | system_message 指令不够清晰 | 优化 `team_manager.py` 中的提示词 |
| 团队状态无法恢复 | 状态文件路径问题 | `team_state.json` 和 `team_history.json` 是相对路径 |

## 八、部署说明

### 8.1 生产环境部署

```bash
# 后端（使用 gunicorn + uvicorn workers）
pip install gunicorn
gunicorn -k uvicorn.workers.UvicornWorker app.main:app --bind 0.0.0.0:8000

# 前端构建
cd frontend
npm run build
# 输出在 frontend/dist/，配置 Nginx 反向代理到后端
```

### 8.2 环境变量配置

```env
# 必须配置
DEEPSEEK_API_KEY=sk-xxxxx

# 可选配置
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-chat
HOST=0.0.0.0
PORT=8000
```

### 8.3 安全注意事项

1. **CORS 配置**：生产环境需将 `allow_origins=["*"]` 修改为具体的域名
2. **API Key 保护**：确保 `.env` 文件不被提交到版本控制（已在 `.gitignore` 中）
3. **文件上传限制**：当前无大小限制，建议添加 `max_size` 参数
4. **SSE 连接安全**：生产环境建议添加认证机制
