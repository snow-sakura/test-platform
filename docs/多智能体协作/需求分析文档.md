# 需求分析文档

## 1. 项目概述

**项目名称**：AutoGen 多代理测试用例协作系统

**项目定位**：一个基于 AutoGen 0.7.5 多代理架构的智能对话应用，通过三个 AI 智能体协作，帮助测试工程师高效地编写和评审测试用例。

**目标用户**：软件测试工程师、QA 团队

---

## 2. 核心需求

### 2.1 多智能体协作（核心业务需求）

**需求描述**：通过多个 AI 智能体分工协作，完成测试用例的编写和评审工作。

| 智能体 | 角色 | 职责 |
|--------|------|------|
| 流程协调员 (coordinator) | 项目经理 | 接收用户需求、分配任务、管理流程、汇总结果 |
| 测试用例编写员 (test_writer) | 执行者 | 编写详细的测试用例（Markdown 表格格式） |
| 测试用例评审员 (test_reviewer) | 质量门禁 | 评审测试用例的完整性、准确性和可执行性 |

**工作流程**：
1. 用户发送需求 → 协调员分配任务
2. 编写员编写测试用例（覆盖正常、异常、边界场景）
3. 评审员评审 → 若拒绝则返回第2步修订，若通过则终止

### 2.2 实时流式通信（非功能性需求）

**需求描述**：AI 回复需要实时流式推送到前端，用户无需等待完整响应。

- 使用 SSE (Server-Sent Events) 协议
- 支持连接状态实时反馈（已连接/处理中/未连接）
- 自动重连机制（最多 5 次尝试）
- 60 秒心跳保活

### 2.3 文件上传支持（功能需求）

**需求描述**：用户可上传文档作为测试用例编写的参考素材。

- 支持的格式：PDF、TXT
- 前端使用 pdfjs-dist 解析 PDF
- 后端使用 PyMuPDF 解析 PDF
- 附件内容自动拼接到用户消息中

### 2.4 会话管理（功能需求）

**需求描述**：每个对话拥有独立会话，支持多轮对话。

- 每个会话具有唯一 session_id（UUID）
- 会话状态会持久化到本地文件（JSON）
- 对话历史也会记录到本地文件

### 2.5 消息展示（UI 需求）

**需求描述**：智能体消息需要有清晰的可视化区分。

- 时间轴线和彩色圆点标识不同智能体
- 每个智能体有独特的颜色和图标
- 长内容（>500 字符）支持折叠/展开
- 消息滑入动画效果

---

## 3. 用户交互流程

```
用户打开页面
    ↓
前端创建会话 (POST /api/session) → 获取 session_id
    ↓
前端建立 SSE 连接 (GET /api/stream/{session_id})
    ↓
用户输入消息 + (可选)附件 → 点击发送 (POST /api/chat)
    ↓
后端启动 AutoGen 多代理对话
    ↓
每个智能体的响应通过 SSE 流式推送 (AgentResponse 事件)
    ↓
前端按智能体分组展示消息（时间轴 + 彩色标识）
    ↓
对话完成 → 用户可继续输入或上传文件进行下一轮
```

---

## 4. 约束条件

- **模型依赖**：必须接入 DeepSeek API（通过 OpenAI 兼容接口）
- **无视觉能力**：DeepSeek 不支持图片识别（model_info.vision = false）
- **最大消息数**：单次对话最多 30 条消息（防止无限循环）
- **终止关键词**：智能体输出 "TERMINATE" 时对话结束
