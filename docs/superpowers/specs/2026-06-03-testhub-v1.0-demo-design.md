# TestPlate V1.0 初版 Demo 设计文档

## 概述

基于 `ai_doc/` 需求文档，开发 TestPlate 测试管理平台 V1.0 基础平台的初版 demo。技术栈为 Django 4.2 + DRF（后端）和 Next.js + Ant Design + Zustand + next-intl（前端）。初版**不包含用户认证模块**。

## 后端设计

### 技术栈
- Django 4.2 + Django REST Framework
- django-filter（搜索/筛选）
- django-cors-headers（跨域）
- SQLite 数据库（初版）

### App 结构
```
backend/
├── config/          # Django 项目配置
│   ├── settings.py  # DRF AllowAny、SQLite、CORS 全放开
│   ├── urls.py      # admin + api 路由
│   └── wsgi.py
├── apps/
│   └── projects/    # 项目管理模块
│       ├── models.py      # Project 模型
│       ├── serializers.py # ProjectSerializer
│       ├── views.py       # ModelViewSet CRUD
│       ├── urls.py        # router.register
│       ├── filters.py     # name搜索 + status筛选
│       └── admin.py       # Django Admin 注册
├── manage.py
└── requirements.txt
```

### Project 模型
| 字段 | 类型 | 说明 |
|------|------|------|
| name | CharField(200, unique) | 项目名称 |
| description | TextField(blank) | 项目描述 |
| status | CharField(20, choices=active/archived/completed) | 状态，默认 active |
| start_date | DateField(null, blank) | 开始日期 |
| end_date | DateField(null, blank) | 结束日期 |
| created_at | DateTimeField(auto_now_add) | 创建时间 |
| updated_at | DateTimeField(auto_now) | 更新时间 |

### API 端点
| 方法 | 路径 | 功能 |
|------|------|------|
| GET | /api/projects/ | 列表（分页+搜索+筛选+排序） |
| POST | /api/projects/ | 创建 |
| GET | /api/projects/{id}/ | 详情 |
| PUT | /api/projects/{id}/ | 全量更新 |
| PATCH | /api/projects/{id}/ | 部分更新 |
| DELETE | /api/projects/{id}/ | 删除 |

## 前端设计

### 技术栈
Next.js 14 (App Router) + Ant Design 5 + Zustand + next-intl + Axios + TypeScript

### 路由设计
| 路径模式 | 页面 | 说明 |
|----------|------|------|
| /[locale]/ | 重定向 → /home | 根页面 |
| /[locale]/home | 首页导航卡片 | 8 张功能卡片 |
| /[locale]/projects | 项目列表 | 搜索/筛选/CRUD |
| /[locale]/projects/[id] | 项目详情 | 项目信息展示 |
| /[locale]/[...rest] | ComingSoon | 未实现模块占位 |
| /[locale]/not-found | 404 | 页面未找到 |

### Layout 布局
- **侧边栏**：Ant Design Sider，深色主题，240px/可折叠，Menu 组件
- **顶栏**：Header 白色 60px，折叠按钮+面包屑+语言切换 Select
- **主内容区**：Content 包裹子页面

### 关键实现
1. **国际化**：next-intl v3，中英文 JSON 语言包，middleware 路由中间件
2. **状态管理**：Zustand + persist（侧边栏折叠、语言偏好）
3. **API 层**：Axios 实例（无 Token），Next.js rewrites 代理 /api 到 Django
4. **UI 组件**：Ant Design Table/Form/Modal/Descriptions/Menu/DatePicker

## 并行开发策略

- **Agent A（后端）**：Django 项目 + Project 模块 CRUD API
- **Agent B（前端）**：Next.js 项目 + Layout + 首页 + 项目管理页面
- 两个代理完全独立，无共享状态或文件冲突（不同目录）
